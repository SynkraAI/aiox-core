# Repertoire Mapper — Knowledge Base

## Domain Overview

O mapeamento de repertórios pessoais é a prática de identificar, extrair,
classificar e organizar todo o conhecimento, habilidades, frameworks mentais,
padrões de decisão e construtos pessoais que uma pessoa acumulou ao longo
de sua vida e carreira.

O desafio central: **a maior parte do repertório é tácito** — a pessoa sabe
mais do que consegue articular (Polanyi, 1966). O repertório mais valioso
(intuição, pattern recognition, julgamento) é exatamente o mais difícil de mapear.

## Key Concepts

### Conhecimento Tácito vs Explícito (Polanyi)

| Tipo | Descrição | Exemplo | Mapeabilidade |
|------|-----------|---------|---------------|
| **Explícito** | Articulável, codificável | Receita de bolo, manual de processo | Alta |
| **Tácito Relacional** | Poderia ser explícito mas não foi | "Eu faço assim, mas nunca escrevi" | Média-Alta |
| **Tácito Somático** | Incorporado no corpo | Andar de bicicleta, tocar piano | Baixa |
| **Tácito Coletivo** | Embutido em práticas sociais | Cultura organizacional, "como fazemos aqui" | Média |

### Periodic Table of Expertises (Collins & Evans)

Níveis de expertise especializada:
1. **Beer-mat knowledge** — Sabe o básico (conversa de bar)
2. **Popular understanding** — Leu livros populares, entende conceitos-chave
3. **Primary source knowledge** — Leu fontes primárias/acadêmicas
4. **Interactional expertise** — Consegue conversar com practitioners
5. **Contributory expertise** — Consegue FAZER o trabalho

### SECI Model (Nonaka & Takeuchi)

Espiral de conversão de conhecimento:
- **S**ocialização (Tácito→Tácito): Aprender por experiência compartilhada
- **E**xternalização (Tácito→Explícito): Articular via metáforas e modelos — **CORE do mapeamento**
- **C**ombinação (Explícito→Explícito): Sistematizar em base de conhecimento
- **I**nternalização (Explícito→Tácito): Aprender fazendo, incorporar

### Deep Smarts (Dorothy Leonard)

6 características de expertise baseada em experiência:
1. **Conhecimento profundo do domínio** — Extensão e profundidade
2. **Reconhecimento de padrões** — Ver o invisível
3. **Perspectiva sistêmica** — Entender interdependências
4. **Consciência de contexto** — Sensibilidade situacional
5. **Acuidade diagnóstica** — Velocidade e precisão
6. **Networking habilidoso** — Saber quem sabe o quê

### Personal Constructs (George Kelly)

- Pessoas entendem o mundo através de **construtos bipolares** que criaram
- Construtos são elicitados por **comparação** (não introspecção)
- O **Repertory Grid** é a ferramenta de extração
- **Laddering** move pela hierarquia: up = valores, down = comportamentos

### Espoused Theory vs Theory-in-Use (Argyris & Schön)

- **Espoused Theory**: O que a pessoa DIZ que acredita/faz
- **Theory-in-Use**: O que o COMPORTAMENTO revela
- **O gap entre os dois** é frequentemente invisível à pessoa
- **Double-Loop Learning**: Questionar as premissas, não só corrigir erros

## Best Practices

### Para Extração

1. **Use incidentes, não perguntas abstratas** (Klein CDM)
   - BOM: "Conte sobre uma vez que você teve que decidir X"
   - RUIM: "Como você toma decisões?"

2. **Use comparação, não introspecção** (Kelly Repertory Grid)
   - BOM: "Destes 3, quais 2 são parecidos? Em quê?"
   - RUIM: "Descreva seu estilo de liderança"

3. **Triangule fontes** — Combine pelo menos 2 métodos
   - CDM para decisões críticas
   - Repertory Grid para construtos pessoais
   - Deep Smarts para priorização

4. **Valide sempre** (Argyris)
   - Compare o que a pessoa diz com evidências de prática
   - Procure contradições — elas são informação valiosa

### Para Organização

1. **PARA para ação, Zettelkasten para conexão**
   - Use PARA (Forte) para organizar por actionability
   - Use linking (Ahrens/Luhmann) para construir grafo

2. **Progressive Summarization** para distilação
   - Não tente resumir tudo de uma vez
   - Múltiplas passadas: Capture → Bold → Highlight → Summary

3. **Uma ideia por nó** no knowledge graph
   - Atômico: cada nó é auto-contido
   - Linkado: conexões explícitas entre nós

### Para Validação

1. **Ladder of Inference** para checar saltos lógicos
2. **Left-Hand Column** para revelar pensamentos não ditos
3. **Pre-Mortem** (Klein) para testar robustez de frameworks

## Common Patterns (RM-Patterns)

### RM-001: Expertise Asymmetry
**O que é:** A pessoa tem expertise contributória em uma área mas apenas
interactional em área adjacente que ela PENSA dominar.
**Sinal:** Confiança alta + vocabulário sofisticado + ausência de exemplos práticos.
**Ação:** Reclassificar para nível correto, usar como oportunidade de growth.

### RM-002: Tacit-Explicit Inversion
**O que é:** O conhecimento mais valioso é o menos articulado.
**Sinal:** "Eu só sei" / "É feeling" / "É experiência" para as decisões mais críticas.
**Ação:** Priorizar CDM extraction nessas áreas.

### RM-003: Construct Rigidity
**O que é:** Construtos pessoais que não mudam mesmo com evidência contrária.
**Sinal:** Mesmos construtos aparecem em TODOS os domínios sem variação.
**Ação:** Usar laddering + double-loop para explorar flexibilidade.

### RM-004: Espoused-Actual Gap
**O que é:** Diferença significativa entre repertório declarado e praticado.
**Sinal:** CDM revela padrões diferentes dos declarados em Q&A.
**Ação:** Apresentar gap com evidência, facilitar reconciliação.

### RM-005: Knowledge Island
**O que é:** Área de expertise desconectada do resto do repertório.
**Sinal:** Nó isolado no knowledge graph, sem cross-references.
**Ação:** Explorar conexões potenciais via Repertory Grid cross-domain.

## Anti-Patterns

### ANTI-001: Inventory Fallacy
Listar tudo que a pessoa sabe sem priorizar ou avaliar profundidade.
→ Use Deep Smarts priority matrix (urgência × unicidade × impacto).

### ANTI-002: Articulation Bias
Mapear apenas o que a pessoa consegue articular, ignorando tácito.
→ Use CDM e Repertory Grid, não apenas entrevistas abertas.

### ANTI-003: Snapshot Thinking
Tratar o mapeamento como foto, não como filme.
→ Repertório muda. Planeje sessões recorrentes.

### ANTI-004: Expert Inflation
Aceitar auto-avaliação de expertise sem validação.
→ Use Collins' Periodic Table + evidências de prática.

### ANTI-005: Jargon Preservation
Manter terminologia acadêmica nos outputs finais.
→ Chin traduz para linguagem do practitioner.

## Regulatory / Ethical Considerations

### Consentimento
- O sujeito deve consentir com o mapeamento
- Definir claramente o que será feito com os resultados
- O sujeito tem direito de revisar e corrigir

### Privacidade
- Repertórios podem conter informação sensível
- Gaps e contradições são especialmente delicados
- Outputs devem ter controle de acesso definido

### Limitações
- Nenhum mapeamento é completo — aceite parcialidade
- Bias do extrator é inevitável — triangule
- O mapa não é o território — o repertório real é sempre mais rico
