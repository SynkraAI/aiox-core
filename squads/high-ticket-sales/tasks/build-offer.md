## Framework Reference

**Framework 1: Irresistible Offer Engine**
- Hormozi Value Equation: `Value = (Dream Outcome × Perceived Likelihood) / (Time Delay × Effort/Sacrifice)`
- Grand Slam Offer: Make offer so good people feel stupid saying no
- Stack logic: Core + bonuses worth > price

## Checklist

### Step 1: Diagnose Current Value Equation (15-20 min)

- [ ] Score Dream Outcome (1-10): How desirable is the end result?
- [ ] Score Perceived Likelihood (1-10): How confident are prospects you can deliver?
- [ ] Score Time Delay (1-10 inverted): How long until they see results? (1 = years, 10 = instant)
- [ ] Score Effort/Sacrifice (1-10 inverted): How hard is it to get results? (1 = extreme effort, 10 = effortless)
- [ ] Calculate current value score: `(Dream × Likelihood) / (Time × Effort)`
- [ ] Identify bottleneck: Which variable is dragging value down?

### Step 2: Dream Outcome Engineering (20-25 min)

- [ ] Define the transformation: "From [current pain] to [desired state]"
- [ ] Quantify the outcome: Revenue increase? Time saved? Status gained?
- [ ] Identify emotional drivers: What does this outcome mean to them? (freedom, respect, security)
- [ ] Articulate the "after" state: Paint vivid picture of life after transformation
- [ ] Test positioning: Does this outcome make them lean forward?

### Step 3: Maximize Perceived Likelihood (15-20 min)

- [ ] List proof elements: Case studies, testimonials, track record
- [ ] Identify credibility boosters: Credentials, media, social proof
- [ ] Add believability mechanisms: Money-back guarantee, results guarantee, conditional offers
- [ ] Address skepticism head-on: "You might be thinking..."
- [ ] Stack trust signals: Authority + social proof + risk reversal

### Step 4: Minimize Time Delay & Effort (20-25 min)

- [ ] Map time to first win: When do they see initial results?
- [ ] Create quick-win milestones: Days 1-7-30-90 roadmap
- [ ] Design "done for you" components: What can you do for them?
- [ ] Remove friction points: Simplify onboarding, provide templates, automate steps
- [ ] Position as accelerator: "We compress 12 months into 90 days"

### Step 5: Offer Stack Architecture (30-40 min)

- [ ] Define core offer: The main product/service
- [ ] Add strategic bonuses: What amplifies the core outcome?
  - [ ] Bonus 1: Addresses time delay (fast-start program)
  - [ ] Bonus 2: Addresses effort (templates/swipe files)
  - [ ] Bonus 3: Addresses likelihood (1-on-1 implementation support)
  - [ ] Bonus 4: Addresses dream outcome (advanced strategy/VIP access)
- [ ] Assign retail value to each component: Be specific ($X,XXX value)
- [ ] Calculate total stack value: Sum all components
- [ ] Design guarantee: Conditional (results-based) or unconditional (money-back)?
- [ ] Add scarcity: Limited spots? Deadline? Bonus expiration?
- [ ] Test the "feel stupid saying no" threshold: Is value 10x+ price?

### Step 6: Naming & Positioning (15-20 min)

- [ ] Create offer name: Outcome-focused, not feature-focused
  - Good: "Revenue Acceleration Program"
  - Bad: "12-Week Coaching Package"
- [ ] Write one-sentence positioning: "[Outcome] for [market] without [pain]"
- [ ] Craft the offer reveal script: How you'll present the stack
- [ ] Design pricing anchors: What makes your price look reasonable?
- [ ] Test verbal presentation: Does it flow naturally?

### Step 7: Quality Check & Refinement (10-15 min)

- [ ] Run through `checklists/offer-quality-checklist.md` (if exists)
- [ ] Verify value equation improvement: New score > old score?
- [ ] Check logical flow: Does each bonus reinforce the others?
- [ ] Validate pricing: Does market support this? Do you believe it?
- [ ] Test objection resistance: Can you defend every component?
- [ ] Finalize `offer-architecture.md` with all components

## Execution Notes

**Duration:** 2-3 hours for complete offer architecture

**Iteration Pattern:**
1. First pass: Get all components on paper
2. Second pass: Refine value equation scores
3. Third pass: Polish naming and positioning

**Common Pitfalls:**
- Weak dream outcome (not specific enough)
- Too many bonuses (dilutes focus)
- Arbitrary pricing (not anchored to value)
- Generic guarantees (not believable)

**Success Criteria:**
- Value equation score > 7.0
- Stack value 10x+ price
- You feel confident presenting it
- Prospect would feel stupid saying no
