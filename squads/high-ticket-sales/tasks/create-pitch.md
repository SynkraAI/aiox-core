## Checklist

### Step 1: Set the Frame (Opening — 5-7 min)

- [ ] Design pattern interrupt: Open with unexpected statement/question
- [ ] Establish authority: Why should they listen to you?
- [ ] State the Big Promise: "By the end, you'll know how to [outcome] without [pain]"
- [ ] Disarm sales resistance: "I'm not here to sell you, but I will make an offer at the end"
- [ ] Create open loop: Tease the One Big Thing you'll teach
- [ ] Test frame strength: Does this make them lean in?

**Frame Control Principles (Klaff):**
- Your frame (expert/prize) > Their frame (skeptical buyer)
- Hot cognition (emotional) > Cold cognition (analytical)
- Status + novelty = attention

### Step 2: Teach One Big Thing (Educational Content — 15-20 min)

- [ ] Identify the "One Big Thing": The core insight that changes everything
- [ ] Structure as story: Origin story → struggle → discovery → breakthrough
- [ ] Introduce the vehicle: The method/system/framework (give it a name)
- [ ] Provide aha moment: "This is why everything else failed..."
- [ ] Give tactical value: Teach something they can use immediately
- [ ] Demonstrate expertise: Show depth without overwhelming
- [ ] Reinforce credibility: Results, case studies, proof

**Education Structure:**
1. The problem (they already know this pain)
2. Why traditional solutions fail (validate their frustration)
3. The new way (your vehicle/method)
4. How it works (simplified explanation)
5. Why it works (logic + proof)

### Step 3: Break & Rebuild Beliefs (15-20 min)

- [ ] Identify 3 False Beliefs blocking action:
  - [ ] **False Belief #1** (about the vehicle): "I don't think [method] works"
  - [ ] **False Belief #2** (about themselves): "I don't think I can do this"
  - [ ] **False Belief #3** (about timing): "I don't think now is the right time"

**For each false belief:**
- [ ] State the belief explicitly: "You might be thinking..."
- [ ] Tell a story that breaks the belief: Epiphany story, case study, origin story
- [ ] Provide logic/data: Back up the story with evidence
- [ ] Reframe the belief: "The truth is..."
- [ ] Transition to next belief: "But even if you believe that..."

**Belief Break Pattern (Brunson):**
- Old belief: "I need [X] to succeed"
- Breaking story: Person without [X] who succeeded
- New belief: "What I actually need is [your vehicle]"

### Step 4: Stack the Value (Offer Reveal — 10-15 min)

- [ ] Transition to offer: "So you're probably wondering how to get started..."
- [ ] Introduce offer name: "[Offer Name] is designed to [outcome]"
- [ ] Present core offer: What they get (be specific)
- [ ] Add bonuses one by one:
  - [ ] Bonus 1: Name + value + why it matters
  - [ ] Bonus 2: Name + value + why it matters
  - [ ] Bonus 3: Name + value + why it matters
  - [ ] Bonus 4+: (if applicable)
- [ ] Announce total value: "If you bought these separately, you'd pay $X,XXX"
- [ ] Create price anchor: "Normally this would be $X,XXX, but..."
- [ ] Reveal actual price: "Today you can get everything for just $X,XXX"
- [ ] Add guarantee: "And you're protected by [guarantee details]"
- [ ] Introduce scarcity: Limited spots, deadline, bonus expiration
- [ ] Calculate value gap: "You're getting $X,XXX value for $X,XXX"

**Stack Logic (Hormozi):**
- Each bonus solves a specific objection
- Total value must be 10x+ price
- Stack builds emotional momentum

### Step 5: Close with Conviction (Final 5 min)

- [ ] Call to action: "Here's what to do next..."
- [ ] Make it dead simple: "Click the button below, fill out the form, join the call"
- [ ] Reinforce urgency: "Remember, this offer expires [when]"
- [ ] Address last objection: "If you're thinking [objection], let me address that..."
- [ ] Future pace: "Imagine 90 days from now when [outcome]..."
- [ ] Final CTA: "Go ahead and [action] right now"
- [ ] Thank and transition: "I'll see you inside" or "Questions? Let's chat"

**Close Principles:**
- Assume the sale (not "if" but "when")
- Make decision easy (remove friction)
- End with certainty (not "maybe" or "hopefully")

### Step 6: Quality Check & Refinement (15-20 min)

- [ ] Read entire script aloud: Does it flow naturally?
- [ ] Check timing: Does each section fit the duration?
- [ ] Verify belief breaks: Do the stories actually break beliefs?
- [ ] Test offer stack: Does value build progressively?
- [ ] Validate frame: Do you maintain authority throughout?
- [ ] Check authenticity: Does this sound like you?
- [ ] Run through `checklists/pitch-quality-checklist.md` (if exists)

## Execution Notes

**Duration:** 3-4 hours for complete pitch script

**Format Adaptations:**

| Format | Adjustments |
|--------|-------------|
| **Webinar (60min)** | Full structure, detailed stories |
| **VSL (20min)** | Compress stories, faster pace |
| **Stage (20-30min)** | More energy, audience interaction |
| **Sales Letter** | Convert to written copy, add subheads |

**Iteration Pattern:**
1. First pass: Get structure and stories on paper
2. Second pass: Refine belief breaks and transitions
3. Third pass: Polish language and timing

**Common Pitfalls:**
- Weak frame (sounds salesy from the start)
- Teaching too much (overwhelming, not one big thing)
- Generic belief breaks (not addressing actual barriers)
- Boring offer reveal (no momentum, flat delivery)

**Success Criteria:**
- Frame feels strong (you're the prize)
- One Big Thing is clear and valuable
- Belief breaks create aha moments
- Offer reveal builds excitement
- You feel confident delivering it
