# ✅ Checkpoints de Progresso - Sistema de Acompanhamento

**Objetivo:** Rastrear progresso do aluno e garantir domínio antes de avançar

---

## 🎯 Como Usar Este Sistema

1. **Antes de começar módulo:** Revisar checkpoints do módulo
2. **Durante o módulo:** Marcar checkpoints conforme completa
3. **Antes de avançar:** Verificar se atingiu mínimo (70% dos checkpoints)
4. **Se travou:** Revisar conceitos do checkpoint que falhou

## 🟢 Módulo 1: Conexão Inicial

### Checkpoint 1.1: Aberturas Criativas ✅
**Critério:** 3 aberturas diferentes criadas (sem currículo)

**Verificação:**
- [ ] Abertura 1 escrita e revisada
- [ ] Abertura 2 escrita e revisada
- [ ] Abertura 3 escrita e revisada
- [ ] Nenhuma menciona currículo
- [ ] Cada uma tem elemento imprevisível claro

**Bloqueadores Comuns:**
- Forçar humor que não é autêntico → Solução: Use suas próprias histórias
- Todas muito parecidas → Solução: Varie formato (piada, dinâmica, truque)

---

### Checkpoint 1.2: Abertura de 60 Segundos ✅
**Critério:** Abertura natural que cabe em 60 seg

**Verificação:**
- [ ] Escolheu 1 das 3 aberturas
- [ ] Praticou 10+ vezes
- [ ] Cabe em exatos 60 segundos
- [ ] Soa natural (não decorada)
- [ ] Vídeo gravado

**Bloqueadores Comuns:**
- Muito longa (>90 seg) → Solução: Cortar detalhes desnecessários
- Parece decorada → Solução: Pratique mais até naturalizar

---

### Checkpoint 1.3: Banco de Humor Autêntico ✅
**Critério:** 3 histórias engraçadas 100% verdadeiras

**Verificação:**
- [ ] 10 histórias listadas
- [ ] 3 selecionadas (mais engraçadas)
- [ ] Escritas em 1 parágrafo cada
- [ ] Todas 100% verdadeiras
- [ ] Pelo menos 1 conecta com tema

**Bloqueadores Comuns:**
- Não acha nada engraçado → Solução: Pergunte a amigos "qual história minha te faz rir?"
- Histórias muito longas → Solução: Foque no momento engraçado específico

---

### Checkpoint 1.4: Feedback de Grupo ✅
**Critério:** 2 de 3 pessoas ficaram curiosas

**Verificação:**
- [ ] Testado com 3-5 pessoas
- [ ] Feedback coletado por escrito
- [ ] Pelo menos 2 ficaram curiosos
- [ ] Abertura ajustada baseado em feedback

**Bloqueadores Comuns:**
- Grupo não deu feedback honesto → Solução: Peça anonimato ou use formulário
- Ninguém ficou curioso → Solução: Revise elemento de imprevisibilidade

---

**Status Módulo 1:** ⬜ 0/4 completos
**Próximo Checkpoint:** 1.1 - Criar 3 aberturas

## 🚨 Sistema de Alertas

### ⚠️ Alerta: Travou em Checkpoint?
**Se você tentou 3x e não passou:**
1. Revise conceito relacionado no módulo
2. Assista exemplo prático (Palestra Romani)
3. Peça feedback de mentor/colega
4. Considere fazer exercício anterior novamente

### ⚠️ Alerta: Pulou Checkpoint?
**Não avance sem completar mínimo (70%):**
- Qualidade > velocidade
- Base fraca = palestra fraca
- Cada checkpoint é fundação para próximo

### ✅ Dica: Celebre Conquistas!
**A cada checkpoint completo:**
- Tire screenshot do progresso
- Compartilhe com grupo de estudo
- Recompense-se (pausa, snack, etc)
- Revise o quanto evoluiu desde Checkpoint 1.1
