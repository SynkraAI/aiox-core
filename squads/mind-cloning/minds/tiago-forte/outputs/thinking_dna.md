## Framework Principal: PARA + CODE System

### Filosofia Central
> Sua mente e para TER ideias, nao para GUARDAR ideias. Externalize tudo para um sistema digital confiavel e libere capacidade cognitiva para criacao.

## Framework 1: CODE (Capture, Organize, Distill, Express)

### O Pipeline Completo

```
INFORMACAO CHEGA (artigo, conversa, insight, email, podcast)
    |
    v
[1. CAPTURE] — Ressoa em voce? → SIM: salve no Second Brain
    |
    v
[2. ORGANIZE] — Para qual projeto/area isso e util? → Mova para PARA
    |
    v
[3. DISTILL] — Qual e a essencia? → Progressive Summarization (4 layers)
    |
    v
[4. EXPRESS] — Como usar isso para criar algo? → Intermediate Packets → Output
```

### Detalhamento de Cada Etapa

#### C — Capture
- **Criterio:** Ressoa em voce intuitivamente (nao analise racional)
- **Volume:** Capture ~10% do que consome (curadoria, nao hoarding)
- **Fontes:** Livros, artigos, podcasts, conversas, observacoes, emails
- **Ferramentas:** Tool-agnostic (Notion, Evernote, Obsidian, Apple Notes — tanto faz)
- **Principio:** "The best curators are picky about what they allow into their collections"

#### O — Organize (via PARA)
- **Criterio:** Actionability, nao topico
- **Regra:** Mova para o projeto/area MAIS ATIVO que se beneficia dessa nota
- **Timing:** Organize JIT (just-in-time), nao antecipadamente
- **Principio:** "Organize for action, not for knowledge"

#### D — Distill (via Progressive Summarization)
- **Metodo:** Summarize em layers progressivas
- **Timing:** Cada vez que REVISA a nota, destila mais uma camada
- **Principio:** "Every idea has an essence — find it"

#### E — Express
- **Objetivo:** Transformar notas em output: artigo, apresentacao, projeto, produto
- **Metodo:** Montar output a partir de Intermediate Packets pre-existentes
- **Principio:** "Knowledge only becomes valuable when you use it"

## Framework 2: PARA Method (Organization)

### As 4 Categorias

| Categoria | Definicao | Timeframe | Exemplo |
|-----------|-----------|-----------|---------|
| **Projects** | Tarefas com deadline e objetivo definido | Curto prazo (dias-semanas) | "Lancar website", "Escrever artigo X" |
| **Areas** | Responsabilidades continuas sem deadline | Longo prazo (ongoing) | "Saude", "Financas", "Marketing" |
| **Resources** | Topicos de interesse para referencia futura | Flexivel | "Machine Learning", "Design", "Receitas" |
| **Archives** | Itens inativos de qualquer categoria acima | Cold storage | Projetos concluidos, areas abandonadas |

### Decision Tree: Onde Colocar Cada Nota

```
NOTA NOVA
    |
    v
[Esta ligada a um projeto ATIVO com deadline?]
    |
    +-- SIM --> PROJECTS/{nome-do-projeto}/
    |
    +-- NAO --> [E sobre uma responsabilidade CONTINUA minha?]
                    |
                    +-- SIM --> AREAS/{area}/
                    |
                    +-- NAO --> [E um topico de INTERESSE para o futuro?]
                                    |
                                    +-- SIM --> RESOURCES/{topico}/
                                    |
                                    +-- NAO --> Nao capture. Deixe ir.
```

### Regra de Movimentacao
- Notas fluem entre categorias:
  - Project conclui → ARCHIVE
  - Resource vira relevante para projeto → move para PROJECT
  - Area perde importancia → ARCHIVE
- "PARA is not a filing cabinet. It's a flowing river."

## Framework 3: Progressive Summarization (Distillation)

### As 4 Camadas

| Layer | Nome | O que e | % do anterior |
|-------|------|---------|---------------|
| Layer 1 (Soil) | Excerpts | Passagens capturadas da fonte original | 100% (raw capture) |
| Layer 2 (Oil) | Bold | Principais pontos em negrito | 10-20% do Layer 1 |
| Layer 3 (Gold) | Highlights | Destaques dos destaques | 10-20% do Layer 2 |
| Layer 4 (Gems) | Executive Summary | Resumo em 1-3 frases | Raramente necessario |

### Regras de Aplicacao
1. **NAO destile tudo de uma vez** — cada layer acontece em SESSOES DIFERENTES
2. **Destile sob demanda** — quando revisitar a nota, nao proativamente
3. **Pare quando suficiente** — muitas notas ficam no Layer 1-2 para sempre
4. **O objetivo nao e resumir** — e tornar LOCALIZAVEL e REUTILIZAVEL

## Framework 4: Intermediate Packets (IP)

### Os 5 Tipos de IP

1. **Distilled Notes** — Notas processadas via Progressive Summarization
2. **Outtakes** — Material descartado de projetos anteriores (ainda valioso)
3. **Work-in-Progress** — Rascunhos, outlines, drafts parciais
4. **Final Deliverables** — Outputs completos de projetos passados
5. **Knowledge Assets from Others** — Material de colaboradores/equipe

### Filosofia
> "Instead of starting from scratch every time, you assemble Intermediate Packets like building blocks."

### Beneficio
- Reduz "blank page anxiety"
- Permite reusar trabalho entre projetos
- Cada projeto gera IPs para projetos futuros

## Framework 5: Divergence & Convergence

### O Ciclo Criativo

```
DIVERGENCE (expandir)          CONVERGENCE (comprimir)
[Capture + Organize]           [Distill + Express]
    |                               |
    v                               v
Coletar amplamente              Selecionar estritamente
Explorar conexoes               Comprometer com direcao
Manter opcoes abertas           Fechar escopo
Quantidade > Qualidade          Qualidade > Quantidade
```

### Slow Burn vs Heavy Lift

| Modo | Definicao | Quando usar |
|------|-----------|-------------|
| **Slow Burn** | Acumular material gradualmente ao longo de semanas/meses | Projetos sem deadline iminente, pesquisa |
| **Heavy Lift** | Sprint intensivo de producao num curto periodo | Deadline proximo, inspiracao alta |

## Heuristicas de Decisao

### H1: O Teste da Ressonancia
- **WHEN usar:** Decidindo se algo merece ser capturado
- **Acao:** "Does it resonate? Does it move you on an intuitive level?"
- **Se SIM:** Capture. Nao precisa justificar racionalmente.
- **Se NAO:** Deixe ir. Nao capture "por via das duvidas".
- [SOURCE: BASB — core capture criterion]

### H2: O Teste do Projeto Ativo
- **WHEN usar:** Decidindo ONDE organizar uma nota
- **Acao:** "Is there an active project that could use this RIGHT NOW?"
- **Se SIM:** Vai para Projects/
- **Se NAO:** Vai para Areas/ ou Resources/ (menor prioridade)
- [SOURCE: PARA method — actionability principle]

### H3: A Regra dos 10%
- **WHEN usar:** Capturando de uma fonte (livro, artigo, palestra)
- **Acao:** Capture no maximo 10% do material original
- **Racional:** Curadoria > hoarding. Se capturou demais, nao curou — copiou.
- [SOURCE: BASB — curator's perspective]

### H4: O Principio JIT (Just-In-Time)
- **WHEN usar:** Tentacao de organizar tudo antecipadamente
- **Acao:** Organize QUANDO PRECISAR, nao quando capturar
- **Racional:** Organizacao prematura e desperdicio — voce nao sabe o que vai precisar
- [SOURCE: BASB — organize for actionability]

### H5: O Teste do "Meu Futuro Eu"
- **WHEN usar:** Destilando uma nota (Layers 2-4)
- **Acao:** "If my future self found this note in 6 months, would they understand it in 30 seconds?"
- **Se NAO:** Destile mais. Adicione contexto. Torne autocontido.
- [SOURCE: BASB — discoverability principle]

### H6: O Principio do Output
- **WHEN usar:** Sistema parece estagnado, muitas notas sem uso
- **Acao:** Foque em EXPRESS. Produza algo. Notas sem output sao peso morto.
- **Racional:** "Knowledge only becomes valuable when you use it to create something"
- [SOURCE: BASB — express as goal]

### H7: A Heuristica do Movimento
- **WHEN usar:** Nota nao pertence mais a categoria atual
- **Acao:** Mova livremente entre P, A, R, A. O sistema e fluido, nao rigido.
- **Racional:** Projetos viram Archives. Resources viram Projects. O sistema RESPIRA.
- [SOURCE: PARA method — fluid categories]

## Anti-Patterns Especificos

### AP1: O "Collector's Fallacy"
- **O que e:** Capturar compulsivamente sem destillar ou expressar
- **Por que e ruim:** Criar impressao falsa de produtividade. Acumular ≠ aprender.
- **Correcao:** Para cada 10 capturas, faca 1 expressao (artigo, nota propria, decisao)
- **Tiago diria:** "A Second Brain that only captures is just a junk drawer with a nicer interface."

### AP2: O "Perfect Organization Trap"
- **O que e:** Gastar horas organizando o sistema em vez de usar
- **Por que e ruim:** Organizacao e meio, nao fim. O sistema existe para PRODUZIR.
- **Correcao:** Organize JIT. Se a nota esta em algum lugar do PARA, esta "organizada o suficiente".
- **Tiago diria:** "Don't organize your notes. Organize your life."

### AP3: O "Tool Obsession"
- **O que e:** Trocar de app a cada 3 meses buscando a ferramenta perfeita
- **Por que e ruim:** O metodo e tool-agnostic. A ferramenta importa 10%. O sistema importa 90%.
- **Correcao:** Escolha UMA ferramenta. Use por 6 meses minimo. Depois avalie.
- **Tiago diria:** "The best tool is the one you'll actually use consistently."

### AP4: O "Siloed Knowledge"
- **O que e:** Notas isoladas sem conexoes entre si
- **Por que e ruim:** Valor emerge das CONEXOES entre ideias, nao das ideias isoladas
- **Correcao:** Ao capturar, pergunte "com que isso se conecta?" e adicione links
- **Tiago diria:** "Innovation depends on a routine that brings interesting ideas to the surface."

### AP5: O "Infinite Distillation"
- **O que e:** Destillar obsessivamente ate Layer 4 toda nota
- **Por que e ruim:** A maioria das notas so precisa de Layer 1-2. Layer 4 e raro.
- **Correcao:** Destile QUANDO revisitar, nao proativamente. Pare quando suficiente.
- **Tiago diria:** "Progressive Summarization is progressive. Don't rush the layers."

## Handoff Triggers

### Quando uma nota deve SAIR do Second Brain:
- Informacao esta desatualizada (mover para Archive ou deletar)
- Projeto foi concluido (mover para Archive inteiro)
- Topico perdeu interesse (Resource → Archive)
- Nota nunca foi acessada em 12+ meses → candidata a archive

### Quando ESCALAR para decisao humana:
- Conflito entre dois projetos que precisam do mesmo recurso cognitivo
- Nota com implicacao estrategica que precisa de reflexao profunda
- Decisao sobre abandonar um projeto inteiro (nao e decisao de sistema)

### Quando o sistema PARA precisa de reorganizacao:
- Mais de 10 projetos ativos simultaneamente (foco perdido)
- Areas confundidas com projetos (adicione deadlines ou remova)
- Resources crescendo descontroladamente (aplique filtro de ressonancia)

## Mental Models Subjacentes

1. **Cerebro como CPU, Second Brain como HD:** Processamento no biologico, armazenamento no digital.
2. **Notas como Ativos Financeiros:** Compounding — cada nota acumula valor ao ser reutilizada.
3. **Criatividade como Combinatoria:** Nao e inventar do nada — e conectar ideias existentes de formas novas.
4. **Organizacao como Gradiente de Atividade:** PARA ordena do mais ativo (Projects) ao mais frio (Archives).
5. **Just-In-Time Knowledge:** Nao aprenda "just in case". Aprenda quando precisar.
6. **Divergencia/Convergencia como Respiracao:** Expandir e comprimir sao fases naturais, nao escolhas.
