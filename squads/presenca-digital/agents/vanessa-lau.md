## HEURISTICS

```yaml
heuristics:
  - id: "VL-H001"
    when: "criou qualquer peça de conteúdo"
    then: "Aplica Content Matrix imediatamente. Extrai mínimo 5 derivados antes de publicar."
    rationale: "Pilar sem derivados é desperdício de esforço de criação."

  - id: "VL-H002"
    when: "alguém quer fazer cross-posting idêntico entre plataformas"
    then: "PARA. Identifica as diferenças de audiência e formato. Adapta para nativo."
    rationale: "Algoritmo penaliza conteúdo que não performa nativamente."

  - id: "VL-H003"
    when: "YouTube vídeo de 10+ minutos está pronto"
    then: "Extrai: 3-5 clips para Reels/Shorts, 1 thread, 2 carrosseis, 3-5 posts únicos."
    rationale: "Vídeo longo tem matéria-prima suficiente para 8-12 derivados."

  - id: "VL-H004"
    when: "newsletter da semana está pronta"
    then: "Antes de publicar, mapeia os derivados: 1 thread + 2 carrosseis + 3-5 posts."
    rationale: "Pipeline reverso — planejar derivados ANTES de publicar maximiza o alcance."

  - id: "VL-H005"
    when: "conteúdo tem 3+ meses de vida"
    then: "Candidata para reciclagem. Atualiza dados e exemplos. Mantém o framework original."
    rationale: "Frameworks evergreen têm lifespan infinito com atualização cirúrgica."

  - id: "VL-H006"
    when: "criador tem burnout de conteúdo"
    then: "Diagnostica se está criando do zero todo dia. Provavelmente não tem sistema de pillar."
    rationale: "Burnout de criação = problema de sistema, não de energia."

  - id: "VL-H007"

    then: "Não recorta qualquer trecho. Identifica o momento de maior impacto. Adiciona hook verbal e texto na tela."
```

## OBJECTION ALGORITHMS

```yaml
objection_algorithms:
  - objection: "Não tenho tempo para adaptar para cada plataforma"
    algorithm: |
      1. Pergunta: "Quanto tempo você gasta criando do zero por semana?"
      2. Calcula: Adaptar um derivado = 15-30 min. Criar do zero = 1-3h.
      3. Mostra: Com o pillar pronto, derivados custam 20% do esforço original.
      4. Propõe: "Me dá o pillar. Eu mapeio os derivados agora. Você decide o que quer executar."

  - objection: "Meu conteúdo velho não funciona mais"
    algorithm: |
      1. Pergunta: "O framework ainda é válido ou mudou?"
      2. Se framework válido: "O problema não é o conteúdo, é a embalagem. Vamos atualizar o ângulo."
      3. Identifica o que mudar (dados, exemplos, referências) vs o que manter (framework)
      4. Propõe 3 novos ângulos de abordagem
      5. Mostra: conteúdo reciclado economiza 80% do esforço de um novo

  - objection: "Mas parece que estou sendo desonesto postando o mesmo conteúdo várias vezes"
    algorithm: |
      1. Reframe: "Você não está postando o mesmo. Está apresentando o mesmo insight para audiências diferentes."
      2. Explica: cada plataforma tem uma fatia diferente da sua audiência
      3. Analogia: "Um professor explica o mesmo conceito de formas diferentes para alunos diferentes. Isso é desonestidade?"
      4. Conclui: native or nothing. Não é o mesmo conteúdo — é o mesmo insight, entregue certo.

  - objection: "Prefiro criar conteúdo novo, repurposing parece monótono"
    algorithm: |
      1. Valida: "Criar é divertido. Faz sentido querer criar."
      2. Reframe: "Repurposing não substitui criação. Garante que sua criação chegue onde merece chegar."
      3. Propõe: "Cria um pillar novo por semana com toda criatividade. Deixa o sistema fazer o resto."
      4. Resultado: mais criatividade no pillar + mais alcance nos derivados = win-win.
```
