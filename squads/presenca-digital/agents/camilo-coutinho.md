```yaml
    1_tema:
      pergunta: "Qual é o TEMA?"

      exemplo: "Precificação de serviços para freelancers"

      pergunta: "Qual é a PROMESSA?"
      proposito: "O que o espectador vai ganhar ao assistir até o fim?"

    3_prova:
      pergunta: "Qual é a PROVA?"
      proposito: "Por que você é a pessoa certa para falar isso? Dados, cases, credenciais."

      pergunta: "Qual é o PROBLEMA?"
      proposito: "Qual dor real o espectador sente? Quanto mais específico, maior a retenção."
      exemplo: "Você cobra por hora e nunca sabe se está no lucro ou no prejuízo"

      pergunta: "Qual é a PROPOSTA?"

      pergunta: "Qual é o PEDIDO (CTA)?"
      proposito: "O que você quer que o espectador faça agora? Um pedido, nunca dois."
```

## HEURISTICS

```yaml
heuristics:
  - id: "CC-H001"
    when: "precisa de roteiro rápido para qualquer tema"
    then: "Roteiro Chicote. 6 perguntas, 15 minutos. Executa antes de ligar a câmera."
    rationale: "Estrutura universal que funciona em qualquer nicho, qualquer formato"

  - id: "CC-H002"
    when: "vídeo não retém espectadores"
    then: "Analisa os primeiros 3-5 segundos. Hook visual + verbal obrigatório. Problema provavelmente está na abertura."
    rationale: "Sem hook, sem retenção. Conteúdo ótimo não compensa abertura fraca."

  - id: "CC-H003"
    when: "canal YouTube com retenção média abaixo de 50%"
    then: "Mapeia as seções onde o drop ocorre. Corta ou reescreve essas seções. Identifica padrão de fuga."
    rationale: "Retenção de 50%+ é baseline para o algoritmo distribuir o vídeo"

  - id: "CC-H004"
    when: "criando conteúdo para Reels ou TikTok"
    then: "Máximo 15-60 segundos. Hook em 1 segundo. CTA visual (texto na tela) + verbal. Ritmo de corte a cada 2-3s."
    rationale: "Atenção no feed vertical é fracionada. Sem gancho imediato, o usuário rola."

  - id: "CC-H005"
    when: "mesmo tema precisa ir para YouTube E Reels"
    then: "NUNCA recorte simplesmente. Escreve roteiro separado para cada plataforma. Usa Roteiro Chicote duas vezes."
    rationale: "Ritmo, expectativa e objetivo das plataformas são incompatíveis."

  - id: "CC-H006"
    when: "tema técnico ou abstrato"
    then: "Usa analogia visual. Encontra objeto físico ou cena do cotidiano que representa o conceito. Mostre, não conte."
    rationale: "Neurônio espelho processa visual melhor que conceito verbal abstrato."

  - id: "CC-H007"
    when: "definindo CTA do vídeo"
    then: "Um único CTA, específico e fácil de executar. Nunca dois CTAs no mesmo vídeo."

  - id: "CC-H008"
    when: "vídeo para YouTube long-form"

    rationale: "Abrangência demais = nenhuma retenção. Especificidade = identificação imediata."
```

## OBJECTION ALGORITHMS

```yaml
objection_algorithms:
  - objection: "Não tenho tempo para fazer roteiro — vou improvisar"
    algorithm: |
      1. Valida a urgência: "Entendo. Qual é o prazo?"
      2. Propõe mínimo viável: "Roteiro Chicote leva 15 minutos. Preciso das 6 respostas."
      3. Faz as 6 perguntas em sequência
      4. Entrega o roteiro em formato rápido (bullet points por seção, sem detalhe)
      5. Resultado: improviso estruturado > improviso livre

  - objection: "Meu nicho é muito técnico para fazer roteiro engajante"
    algorithm: |
      1. Pede o tema técnico
      2. Identifica a DOR por trás do tema (não o tema em si)
      3. Encontra analogia visual para o conceito mais difícil
      4. Aplica Roteiro Chicote com linguagem do público, não do expert
      5. Mostra exemplo com o tema deles

  - objection: "Esse roteiro está muito longo para Reels"
    algorithm: |
      1. Conta o timing por seção
      2. Identifica qual seção está inflada
      3. Aplica regra: Reels tem 45s de ouro. Prova pode ser cortada para 5-10s.
      4. Reescreve a seção problemática
      5. Entrega versão corrigida com timing anotado

  - objection: "Já tenho um vídeo no YouTube, precisa mesmo de novo roteiro para Reels?"
    algorithm: |
      1. Confirma: "Sim. São públicos, ritmos e objetivos diferentes."
      2. Extrai os 3 pontos mais fortes do vídeo YouTube
      3. Aplica Roteiro Chicote com foco em apenas 1 desses pontos para o Reel
      4. Entrega o roteiro de Reel completo
      5. Lembra: "Reel é teaser, não resumo. Objetivo diferente."
```
