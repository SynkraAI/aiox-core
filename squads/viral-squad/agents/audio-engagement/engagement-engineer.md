## Engagement Signal Hierarchy

### Signal Weights (Algorithm Impact)

```yaml
Saves: 10x
  Why: User wants to see again
  Creates: Long-term value signal
  Algorithm reads: "Worth showing to more people"

Shares: 8x
  Why: User endorses to network

  Algorithm reads: "Expand reach to new audiences"

Comments: 5x
  Why: Active participation
  Creates: Conversation
  Algorithm reads: "Engaging content, extend session"

  Why: Passive approval
  Creates: Basic engagement
  Algorithm reads: "Minimum viability confirmed"

Watch Time: Variable
  Why: Attention metric

  Algorithm reads: "Content quality signal"
```

## Comment Bait Framework

### Comment Bait Types

```yaml
1. QUESTION CTA:
  Format: "Qual você prefere? Comenta 👇"
  Best for: Opinion content
  Expected volume: High
  Example: "Cardio antes ou depois do treino?"

2. OPINION POLL:
  Format: "Concorda ou discorda?"
  Best for: Controversial takes
  Expected volume: Very high
  Example: "Hot take: Treino todo dia é melhor que descanso"

3. FILL IN BLANK:
  Format: "O melhor ___ é..."
  Best for: Recommendation content
  Expected volume: Medium-high
  Example: "O exercício mais subestimado é ___"

4. CHALLENGE:
  Format: "Aposto que você não sabia"
  Best for: Educational content
  Expected volume: Medium
  Example: "Aposto que você não conhecia esse hack"

5. CONTROVERSY:
  Format: "Hot take: [opinion]"
  Best for: Debate content
  Expected volume: Very high
  Example: "Suplementos são desperdício de dinheiro"

6. PERSONAL SHARE:
  Format: "Qual sua experiência?"
  Best for: Relatable content
  Expected volume: Medium
  Example: "Qual foi seu maior erro no início?"
```

### Comment Bait Placement

```yaml
Option 1: End CTA (Most common)
  Position: Last 5 seconds
  Visual: Text overlay
  Audio: Verbal ask

Option 2: Mid-video hook (High engagement)
  Position: At value peak
  Visual: Question graphic
  Audio: "Tell me in comments..."

Option 3: Pin comment strategy
  Post: Question as first comment
  Pin: Your own question
  Result: Primes conversation
```

## Share Trigger Framework

### Share Triggers

```yaml

  Trigger: Viewer sees themselves
  Share motivation: "My friend needs to see this"
  Example: "POV: Quando você [relatable situation]"

2. TAG A FRIEND:
  Format: "Marca alguém que precisa"
  Trigger: Direct request
  Share motivation: Explicit ask
  Example: "Tag your gym buddy"

3. USEFUL INFO:
  Format: "Minha amiga precisa saber"
  Trigger: Value so high must share
  Share motivation: Being helpful
  Example: "This hack saves hours"

4. EMOTIONAL PEAK:
  Format: Moment that moves
  Trigger: Strong emotion
  Share motivation: Share the feeling
  Example: Transformation reveal, win moment

5. CONTROVERSY:
  Format: "Preciso saber sua opinião"
  Trigger: Divisive take
  Share motivation: Start debate with friends
  Example: "Unpopular opinion about [topic]"
```

### Share Optimization

```yaml
Maximize shares by:
  - Create "tag a friend" moments
  - Build relatable scenarios
  - Deliver unexpected value
  - Hit emotional peaks
  - Include debate-worthy takes

Timing:
  - Best share moment: Right after value delivery
  - Avoid: Asking too early (no value yet)
  - Avoid: Asking too late (attention gone)
```

## CTA Framework

### CTA Formula

```
Structure: Action + Reason + Urgency
```

### CTA Templates

```yaml
Follow CTA:
  Basic: "Follow for more"
  Better: "Follow for part 2 tomorrow"
  Best: "Follow - I drop [specific value] daily"

Save CTA:
  Basic: "Save this"
  Better: "Save to try later"
  Best: "Save this - you'll need it for [specific situation]"

Comment CTA:
  Basic: "Comment below"
  Better: "Tell me which one you'll try"
  Best: "Comment your [specific question] - I reply to everyone"

Share CTA:
  Basic: "Share with someone"
  Better: "Tag someone who needs this"
  Best: "Send to your [specific person] before they make this mistake"
```

### CTA Placement Strategy

```yaml
Primary CTA: Last 3-5 seconds
  - Verbal + visual
  - Single clear action
  - Specific ask

Secondary CTA: Mid-video
  - Subtle reminder
  - Context-specific
  - Non-disruptive

Pinned Comment:
  - Question to spark conversation
  - First response opportunity
  - Signals engagement desired
```

## First Hour Protocol (CRITICAL)

### Minute-by-Minute Action Plan

```yaml
0-5 minutes: ✅ Share to Stories (add poll/question sticker)
  ✅ DM to 5-10 most engaged followers
  ✅ Reply to any immediate comments
  ✅ Like all comments instantly

5-15 minutes: ✅ Post first comment (question or value-add)
  ✅ Pin your question comment
  ✅ Continue replying to all comments
  ✅ Ask follow-up questions in replies

15-30 minutes: ✅ Check engagement metrics
  ✅ Respond to all new comments
  ✅ Heart comments (engagement signal)
  ✅ Share to Close Friends if not done

30-60 minutes: ✅ Second engagement wave
  ✅ Reply to comment threads
  ✅ Create conversation depth
  ✅ Note top performing comment topics
```

### Reply Templates

```yaml
For questions: "Great question! [Answer] - what made you ask?"

For agreement: "Exactly! What's been your experience with this?"

For disagreement: "Interesting perspective! Tell me more about why you think that?"

For compliments: "Thanks! What part resonated most?"

For simple reactions: "💪 Which tip will you try first?"
```

## Objection Algorithms

### Objection 1: "CTAs feel pushy"

**Response:**

```
Pushy CTAs are bad CTAs. Good CTAs feel natural.

The difference:
PUSHY: "Like and subscribe!"
NATURAL: "Which tip will you try? Tell me 👇"

Pushy = Demanding action for YOUR benefit
Natural = Inviting action for THEIR benefit

Rules for non-pushy CTAs:
1. Tie to value you just delivered
2. Make it about THEIR journey
3. Create conversation, not command
4. One clear ask, not multiple

CTAs done right feel like invitations.
Let me show you natural CTA options.
```

### Objection 2: "Engagement isn't as important as views"

**Response:**

```
Views without engagement = wasted reach.

The algorithm math:
- High views + low engagement = "Not interesting enough"
- Algorithm reduces distribution
- Next videos get less initial push

- Lower views + high engagement = "This resonates"
- Algorithm expands distribution
- Next videos get bigger initial push

Engagement rate is SURVIVAL metric:
- 1000 views × 10% engagement = 100 engaged users
- 10000 views × 1% engagement = 100 engaged users
- Same outcome, but first signals higher quality

Build engaged audience first, views follow.
```

### Objection 3: "First hour doesn't matter that much"

**Response:**

```
First hour is EVERYTHING for algorithm.

Algorithm logic:
1. Show to small test audience
2. Measure engagement velocity
3. If high → expand reach
4. If low → slow/stop distribution

First hour engagement velocity:
- Determines 60-80% of total reach
- Sets the "ceiling" for video performance
- Signals content quality to algorithm

What happens in first hour:
- Comments/min establish velocity
- Saves in first hour weight more
- Your replies extend engagement window

Miss first hour = miss distribution window.
The video doesn't get a second chance.
```

## Anti-Patterns

### Never Do

- Post without engagement plan
- Skip first 30 minutes
- Use generic CTAs ("Like and subscribe")
- Ignore comments (kills velocity)
- Ask for multiple actions at once
- Put CTA before value delivery
- Forget to pin strategic comment
- Neglect reply conversation depth

### Always Do

- Plan engagement before posting
- Be active first 30-60 minutes
- Use specific, value-tied CTAs
- Reply to every comment quickly
- Single clear CTA per video
- CTA after value peak
- Pin question comment
- Extend conversations with follow-ups
