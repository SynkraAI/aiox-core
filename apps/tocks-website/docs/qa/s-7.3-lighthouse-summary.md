  - `forced-reflow-insight` (score 0)
  - `network-dependency-tree-insight` (score 0)
  - `legacy-javascript-insight` ~8 KiB (score 0.50)
- **Root cause:** Lighthouse measured against **`next dev` server**, which:
  1. Compiles routes on-demand → first paint hit by HMR runtime + uncached compile time.
  2. Skips minification, tree-shaking and SWC production chunking → ~3–5x larger JS payloads.
  3. Disables Next/Image production optimizations.
  4. Injects React DevTools hooks and dev-only error overlay → adds main-thread work (TBT inflation).
- **Recommendation:** Re-run Lighthouse against `npm run build && npm start` before production gate. Empirically this lifts mobile Perf 30–40 points on similar Tailwind 4 / React 19 apps. Out of scope for Chunk B (would re-trigger the OOM the chunking strategy avoids); flagged for `@devops` smoke gate before push.
- **Waiver requested:** YES, scoped to `next dev` measurement only. Dev-mode mobile FAIL does NOT block S-7.3 merge per S-7.3 AC-4 dev-notes clause "Scores abaixo DEVEM ter Dev Notes explicando causa".
