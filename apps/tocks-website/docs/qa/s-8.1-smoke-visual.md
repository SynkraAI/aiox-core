## Incident noted during gate (resolved before verdict)

**First harness run returned empty-string tokens and Times-New-Roman body** — diagnosed as stale `.next` production-build directory (served CSS chunk `0xcpctd2goja0.css` returned 500; real CSS chunk on disk was `0duic0y-en~s9.css`). `.next/dev` still contained pre-S-8.1 Cormorant/Montserrat font artifacts. **Resolution:** `rm -rf .next && npm run build && npm run start`. Re-run against clean build produced the PASS numbers above. No code fix needed — build hygiene only.

**Action for @devops/@dev:** when verifying S-8.1 changes locally, always clean `.next` before `npm run build` — stale dev artifacts can serve broken prod chunks.
