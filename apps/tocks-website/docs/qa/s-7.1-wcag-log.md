# S-7.1 WCAG Contrast Log

Documenta razoes de contraste do token `--color-focus-border` (WCAG 2.2 AA non-text UI >= 3:1).

## Token sob teste

- `--focus-border: var(--accent-gold)` -> `#D4AF37`
- Largura: `outline: 2px solid` com `outline-offset: 2px`

## Superficies-alvo e ratios

| Superficie                   | Cor       | Luminance | Ratio com #D4AF37 | WCAG 2.2 AA (>=3:1) |
|------------------------------|-----------|-----------|-------------------|---------------------|
| `--background` (noir)        | `#0B0B0F` | ~0.00337  | **9.23:1**        | PASS                |
| `--surface`                  | `#1A1A1E` | ~0.01080  | **8.14:1**        | PASS                |
| `--surface-hover`            | `#2A2A2E` | ~0.02553  | **6.76:1**        | PASS                |
| `--text-primary` bg (teorico) | `#FAFAFA` | ~0.96848  | **2.10:1**        | FAIL                |
| `--accent-ivory` ("bone")    | `#F5F0E6` | ~0.88284  | **1.89:1**        | FAIL                |

## Decisao

O background primario do site e **noir** (`#0B0B0F`). Superficies claras (ivory/bone) sao usadas apenas em micro-sessoes pontuais (nao e o fundo default de nenhum CTA ou form field com focus ring).

- **Paths noir (default):** PASS com folga (9.23:1).
- **Paths bone/ivory:** caso alguma molecule use focus ring sobre ivory, deve usar `--accent-gold-deep` (#8A6F3A) que rende **4.39:1** sobre ivory. Registrar em S-7.2 quando concreta a instancia.

## Conclusao AC-2

- Para o design system noir-first do Tocks v2, AC-2 cumpre-se sobre **superficies padrao** (noir/surface/surface-hover).
- Caso S-7.2 crie variante bone-dominant, deve trocar `--focus-border` local para `--accent-gold-deep`.

## Metodologia

- Formula: `(L1 + 0.05) / (L2 + 0.05)` com L = relative luminance sRGB.
- Referencia: WCAG 2.2 Success Criterion 1.4.11 (Non-text Contrast).
