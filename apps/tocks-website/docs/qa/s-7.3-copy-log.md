> **F-SA-01 (NEW finding for @dev — surfaced during Chunk C smoke):**
> `src/app/actions/submit-concierge.ts` is marked `'use server'` and exports two things:
> 1. `submitConcierge` (async function — OK)
> 2. `INITIAL_CONCIERGE_STATE` (plain object — REJECTED by Next 16 / React 19)
>
> When a user submits the concierge form, the runtime returns HTTP 500 with payload:
> ```
> Error: A "use server" file can only export async functions, found object.
> ```
> See https://nextjs.org/docs/messages/invalid-use-server-value
>
> **Impact:** the form is currently non-functional in production. Submit always 500s. The redirect to wa.me never fires. This is a P0 fix for @dev before any push.
>
> **Suggested fix (NOT applied — out of QA scope):**
> Move `INITIAL_CONCIERGE_STATE` and `ConciergeFormState` to a sibling non-action module (e.g. `src/app/actions/submit-concierge.types.ts`) and re-import from both the action and the form. Roughly 6 lines moved, 2 import edits.
>
> The smoke test is intentionally written so it will **continue to pass** after the fix — both branches (known-failure and post-fix) are encoded.
