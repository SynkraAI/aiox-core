## Incident noted and resolved during gate

**Symptom:** First harness run returned empty-string for all CSS tokens and body rendered in Times New Roman.

**Root cause:** Stale `.next` production-build directory. The served HTML referenced CSS chunk `0xcpctd2goja0.css` (from a prior build hash), but the real chunk on disk was `0duic0y-en~s9.css`. Request for the stale filename returned HTTP 500 Internal Server Error. `.next/dev` still held Cormorant_Garamond and Montserrat font CSS from a pre-S-8.1 dev session.

**Resolution:** `rm -rf .next && npm run build && npm run start`. Clean rebuild served the correct CSS chunk with all tokens and Fraunces/Inter/Space Grotesk font-face declarations. All gate checks then passed.

**Recommendation for @devops/@dev:** treat this as a build-hygiene note. Any local verification of token/font-stack changes should clean `.next` before `npm run build`. No code change in S-8.1 required.
