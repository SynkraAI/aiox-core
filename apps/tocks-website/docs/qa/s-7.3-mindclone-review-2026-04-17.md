## 2. Tensions matrix

### Where they AGREE (3/3 consensus)
1. **Bundle removal of framer-motion = correct decision.** None recommended reverting Patch #2b. Perf gains preserved.
2. **Patch #1 (no animation-delay on H1) = correct.** All three accept H1-as-LCP requires immediate paint.
3. **Current implementation underbakes the choreography.** Same-keyframe + uniform 24px + linear delays = "engineering defaults", not luxury craft.
4. **Solution lives in CSS-only refinement.** No clone wants framer-motion back. The fix is 1 incremental patch (~12-30 lines CSS).
5. **Subtitle/CTA need differentiation from H1.** Either via distance (val-head, tobias) or curve (don-norman, val-head) or scale settling (tobias).

### Where they DISAGREE
| Topic | don-norman | tobias-van-schneider | val-head |
|-------|------------|----------------------|----------|
| **Optimal translateY for H1** | 32px (motion threshold) | 16px (paint speed > motion) | 16px (LCP minimization) |
| **Optimal translateY for subtitle** | 32px | 28px | 32px |
| **Scroll indicator delay** | EARLIER (350-500ms — guides eye-flow) | KEEP/SLOW (1800ms duration) | EARLIER (1500ms delay, 1800ms duration) |
| **Need empirical validation?** | YES (5-second test mandatory) | NO (slow-mo video review enough) | NO (devtools accessibility check enough) |
| **Add scale to subtitle?** | Not mentioned | YES (`scale 0.985 -> 1`) | YES (`scale 0.992 -> 1`) |

### Friction zone (the synthesis call)
**don-norman wants 32px on H1; val-head wants 16px on H1.** Don frames it as motion perception threshold (Wertheimer); val frames it as LCP optimization. Both are right in their domain. **Synthesis decision: 16px on H1 (val wins on LCP — that's the original perf goal we're protecting), 32px on subtitle (Don wins on perception — H1 doesn't need to "move" to be seen, subtitle does).**

**Tobias wants scale 0.985 -> 1 on subtitle; val wants 0.992 -> 1.** Both are micro-settles. **Synthesis: 0.99 -> 1** (midpoint, perceptually equivalent, simpler authoring).

## 4. Patch #2c — exact CSS spec (proposed for @ux-design-expert ratification)

> Replaces lines 151-179 of `src/app/globals.css`. ~30 lines CSS, +0 bytes bundle (gzip absorbs), 0ms LCP delta (transform+opacity = compositor-only), addresses accessibility blocker.

```css
/* Hero entrance choreography -- Mind Clone synthesis (S-7.3 review 2026-04-17).
   Source: docs/qa/s-7.3-mindclone-review-2026-04-17.md
   Driving clones: val-head (timing/curves), tobias-van-schneider (craft), don-norman (validation). */

/* H1 + label -- LCP elements. Micro-translate (16px) preserves paint speed. */
@keyframes fade-up-h1 {
  from { opacity: 0; transform: translate3d(0, 16px, 0); }
  to   { opacity: 1; transform: translate3d(0, 0, 0); }
}
.hero-h1, .hero-label {
  animation: fade-up-h1 700ms cubic-bezier(0.16, 1, 0.3, 1) both;
  will-change: opacity, transform;
}

/* Subtitle -- larger travel + scale settling (luxury "deceleration"). */
@keyframes fade-up-subtle {
  from { opacity: 0; transform: translate3d(0, 32px, 0) scale(0.99); }
  to   { opacity: 1; transform: translate3d(0, 0, 0) scale(1); }
}
.hero-subtitle {
  animation: fade-up-subtle 800ms cubic-bezier(0.16, 1, 0.3, 1) 250ms both;
  will-change: opacity, transform;
}

/* CTAs -- mid travel, longest delay (settling crescendo per val-head). */
@keyframes fade-up-cta {
  from { opacity: 0; transform: translate3d(0, 20px, 0); }
  to   { opacity: 1; transform: translate3d(0, 0, 0); }
}
.hero-ctas {
  animation: fade-up-cta 700ms cubic-bezier(0.16, 1, 0.3, 1) 550ms both;
  will-change: opacity, transform;
}

/* Scroll indicator -- earlier delay, slower duration (premium = unhurried). */
@keyframes fade-in-scroll {
  from { opacity: 0; }
  to   { opacity: 1; }
}
.hero-scroll-indicator {
  animation: fade-in-scroll 1800ms ease-out 1500ms both;
  will-change: opacity;
}

/* BLOCKER FIX -- prefers-reduced-motion (val-head: "accessibility is non-negotiable"). */
@media (prefers-reduced-motion: reduce) {
  .hero-h1, .hero-label, .hero-subtitle, .hero-ctas, .hero-scroll-indicator {
    animation: none;
    opacity: 1;
    transform: none;
  }
}
```

The current state (Patches #1/#1b/#2b shipped) achieves the perf goal but ships a **template-grade** entrance for a luxury brand. All three Mind Clones converged on the same conclusion via different paths: the choreography needs craft, not engineering defaults. Plus there's an **accessibility BLOCKER** (no `prefers-reduced-motion`) that should have been caught in the original review.

**Specific asks for ratification:**
1. **Approve PRESERVE on Patches #1 and #2b** (perf wins, server component, bundle reduction — keep).
2. **Approve ALTER on Patch #1b + apply Patch #2c** as specified above (sibling keyframes, exponential curve, scale settling, redistributed delays).
3. **Mandate the accessibility block** (`prefers-reduced-motion`) — non-negotiable per val-head; should be checklist item for ALL future motion specs.
4. **Optional, post-ship**: Run don-norman's 5-second test (5 users on Maze, "What would you pay for this brand?") to validate the status-signal hypothesis. Time: 30 min. Cost: ~$50.
5. **Hand-off back to @dev** with Patch #2c spec (this document section 4) for implementation.

**What this review accomplished that the previous session didn't:**
- Surfaced the `prefers-reduced-motion` blocker (no core agent caught it)
- Identified that "uniform keyframes" reads as template, not craft (a brand-level concern)
- Provided 3 independent expert lenses (cognitive, visual, motion) instead of single-point review
- Produced a concrete, testable patch spec instead of just critique

The friction was the point. tobias and val both wanted 16px on H1; don-norman wanted 32px. Working through *that disagreement* is how we got to the right answer (16 on H1 because LCP wins; 32 on subtitle because perception wins). That tension was absent in the original session and is what restored signal here.
