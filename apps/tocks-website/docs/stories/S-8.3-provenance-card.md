# Story 8.3 — Provenance Card: Dossier Arquivístico Tufteano

## Story

**COMO** dev agent do Tocks Website v2
**QUERO** criar o organism `provenance-card.tsx` (dossier arquivístico tufteano com origem madeira + timeline criação + desenho técnico isométrico + contador de série) + extender `Product` type com `ProductProvenance` + integrar em `ProductLayout`
**PARA** que cada product page entregue o **cherry-pick E** (Provenance Data-rich) da direção aprovada — diferenciação vs concorrentes brasileiros de móveis luxo, data-ink ratio máximo (Tufte), e storytelling de proveniência ("cada peça tem um nome, uma data, uma procedência").

## Context

- **Direção aprovada:** B (Gilded Noir Cinemático) + **cherry-pick E** → cada product page ganha mini-dossier Tufteano abaixo dos specs.
- **Regra Tufte (ratified consultation ccb3c0a4):** data-ink ratio máximo. Sem grade. Sem fundo decorativo. Tipografia carrega hierarquia. Small multiples para origem madeira + timeline.
- **Spec campos** (master-plan Fase 4a):
  - **Origem madeira:** nome (ex: "Peroba Rosa") + região Brasil (ex: "MT — Mato Grosso")
  - **Ano criação:** timeline horizontal com marcos (desenho → corte → acabamento → entrega)
  - **Peças criadas:** contador "03 de 08 da série"
  - **Dimensões:** desenho técnico isométrico SVG (small, monoline, gold stroke)
  - **Acabamento:** swatch código (usa swatch-picker existente)
- **Product type atual (`src/data/products.ts`):** tem `specs`, `features`, `customizationOptions` — precisa extender com `provenance`.
- **Inventário Art. IV (No invention):** 8 produtos reais (`tenro-luxo`, `gabe`, `ark`, `vertice`, `curve`, `elipse`, `nobus`, `rustic`). Dados provenance **devem ser inventados com bom senso** (origem madeira plausível Brasil, timeline plausível atelier artesanal, série 1-8 honesta para atelier sob medida) — **mas documentar em PR que são placeholders Fase 8**, aguardando validação do atelier real. Sem provenance-card sem dados, story entrega shell + placeholder honesto.
- **Art. VII:** provenance-card <100 linhas. Se extrapolar, split em `provenance-card.tsx` (wrapper) + sub-atoms `provenance-field.tsx` + `provenance-timeline.tsx` + `provenance-isometric.tsx`.
- **Contexto visual:** card renderiza sobre `--bone-paper` (editorial surface). Regra gold-on-bone (S-8.1 AC-9): gold apenas em hairlines/strokes/bullets. Texto em `--bone-ink`.

## Acceptance Criteria

1. **AC-1 (Type ProductProvenance):** GIVEN `src/data/products.ts`, WHEN editado, THEN exporta `interface ProductProvenance { woodOrigin: { species: string; region: string }; timeline: { designed: string; milled: string; finished: string; deliveredEst: string }; serieNumber: { current: number; total: number }; isometricSvgPath?: string }`. Interface `Product` ganha `provenance: ProductProvenance`.

2. **AC-2 (Dados 8 produtos):** GIVEN cada um dos 8 produtos em `PRODUCTS`, WHEN inspecionado, THEN possui campo `provenance` preenchido com **placeholders plausíveis documentados**:
   - `woodOrigin.species`: madeira brasileira plausível (Peroba Rosa, Ipê, Cumaru, Freijó, Sucupira, Jatobá, Imbuia, Cedro — uma por produto, sem duplicatas)
   - `woodOrigin.region`: estado Brasil plausível ("MT", "RO", "PA", "BA", "SC", "MG", "SP", "RS")
   - `timeline`: 4 datas plausíveis (formato "Abr 2024" → "Set 2024" → "Dez 2024" → "Mar 2025") — séries históricas OK
   - `serieNumber.total`: 3-8 (atelier artesanal, séries curtas)
   - `serieNumber.current`: número dentro do range
   - Comentário explícito no arquivo: `// NOTE S-8.3: provenance placeholders — validar com atelier antes de produção`

3. **AC-3 (Organism provenance-card):** GIVEN `src/components/organisms/provenance-card.tsx`, WHEN renderizado com prop `{ provenance: ProductProvenance, productName: string }`, THEN renderiza 4 blocos em grid editorial (Tufte data-ink):
   - **Bloco 1 Origem:** label "Origem" (Fraunces uppercase `--tracking-editorial`) + species+region em linha + small multiple mapa Brasil SVG estilizado com estado destacado gold-700
   - **Bloco 2 Timeline:** label "Criação" + 4 dots ligados por linha horizontal `--bone-rule`, cada dot com data abaixo (Space Grotesk 12px)
   - **Bloco 3 Série:** label "Série" + numeral grande Fraunces `.heading-display`-like "03/08"
   - **Bloco 4 Dimensões:** label "Dimensões" + SVG isométrico monoline stroke `--gold-700` 1px + dimensões `specs.dimensions` abaixo
   - Card bg `--bone-paper`, border-top `--bone-rule` 1px, sem box-shadow, padding-block 4rem.
   - Arquivo <100 linhas.

4. **AC-4 (SVG isométrico):** GIVEN bloco 4, WHEN renderizado, THEN SVG isométrico genérico (representa volume retangular da mesa/pebolim) stroke monoline `--gold-700`, fill-none, viewBox adequado. **Um único SVG shared** para categoria `bilhar` e outro para `pebolim` — **não inventar 8 SVGs únicos** (Art. IV: bom-senso, isometric shared = honestidade visual, não fake customização).

5. **AC-5 (Mapa Brasil SVG):** GIVEN bloco 1, WHEN renderizado, THEN SVG simplificado Brasil (contorno monoline) com estado da `woodOrigin.region` destacado `fill: var(--gold-500)`, restante `stroke: var(--bone-rule)`. SVG armazenado em `public/icons/brazil-map.svg` OU inline component.

6. **AC-6 (WCAG contraste):** GIVEN todos textos do card sobre `--bone-paper`, WHEN medido, THEN:
   - `--bone-ink` sobre `--bone-paper` = 15.2:1 ✅
   - Gold `--gold-700` sobre `--bone-paper` = 4.39:1 ✅ (hairlines/strokes decorativos, NÃO texto — S-8.1 regra)
   - Zero texto gold sobre bone (regra S-8.1)

7. **AC-7 (Integração ProductLayout):** GIVEN `src/components/templates/product-layout.tsx` (ou equivalente que renderiza product page), WHEN editado, THEN renderiza `<ProvenanceCard provenance={product.provenance} productName={product.name} />` **abaixo** do bloco de specs, **acima** de CTA concierge. Section wrapper usa `.section-bone` (S-8.1) + `.container-editorial`.

8. **AC-8 (SSR-safe):** GIVEN `provenance-card.tsx`, WHEN importado em Server Component, THEN **não precisa** `'use client'` (componente visual puro, zero hooks, zero interatividade — apenas SVG inline + texto).

9. **AC-9 (Responsive):** GIVEN viewport <768px, WHEN card renderiza, THEN 4 blocos empilham single-column. Grid `md:grid-cols-2 lg:grid-cols-4`. Timeline bloco 2 mantém scroll horizontal OK se estreito.

10. **AC-10 (axe + Art. VII):** GIVEN card renderizado em product page, WHEN `axe-core` roda, THEN 0/0/0/0. `wc -l src/components/organisms/provenance-card.tsx` < 100 linhas. Se sub-atoms necessários, cada um também <100.

11. **AC-11 (Typecheck + products.ts):** GIVEN `npm run typecheck`, WHEN executado, THEN 0 erros. `Product` interface extension não quebra consumers existentes (ProductCard, ProductLayout, getProductBySlug).

### Restrições críticas

- **Art. IV (No invention):** 8 produtos reais. Não inventar produto 9º. Provenance placeholders documentados como tal — honestidade operacional, não simulação de autenticidade.
- **Art. VII:** card + sub-atoms <100 linhas cada.
- **Regra gold-on-bone (S-8.1 AC-9):** zero texto gold. Apenas strokes/hairlines/bullets decorativos.
- **Tufte data-ink:** sem grade, sem card shadow, sem background decorativo além de `--bone-paper`. Tipografia é o storytelling.
- **SSR-safe:** zero hooks no card. Data vem via prop (server-rendered).

### Tufte ratification (consultation ccb3c0a4)

- 3 aplicações data-ink no site (spec requisitado):
  1. **Product provenance card** (este S-8.3)
  2. Atelier longform small multiple "origem madeira" sidebar (S-8.4)
  3. Opcional: filtro de coleção por madeira/ano (futuro, não S-8)
- **Regra Tufte pura:** todo pixel que não carrega dado é dívida visual. Card não tem box-shadow, não tem border-radius, não tem icon decorativo.

### Spiekermann ratification (indireto via S-8.1)

- Numeral Série "03/08" em Fraunces display weight 300 opsz 144 — tabular-nums para alinhamento.

### Decisão PM sobre mapa Brasil (trade-off)

- **Opção A (rico):** SVG Brasil completo 27 estados → ~15KB gzipped → fiel Tufte small multiple.
- **Opção B (minimal):** retângulo Brasil + dot na região → ~500 bytes → honestidade minimalista.
- **Decisão PM:** @dev escolhe baseado em budget. **Default Opção B** — fiel ao princípio "less ink, more info". Se @dev tem SVG Brasil <20KB pronto, Opção A. Decisão documentada em PR.

### Input para S-8.6 QA

- Provenance card ganha validação no 5-second test: arquiteto olhando card 5s deve captar "madeira certificada + atelier autoral + série limitada". Se falhar, copy placeholder precisa melhorar antes do go-live.

### Riscos identificados

- **R-1 (alto):** dados provenance são fictícios. Se PR for merged sem validação do atelier real, vira content debt. Mitigação: NOTE comment explícito + QA task de validação pré-go-live em S-8.6.
- **R-2 (médio):** Mapa Brasil SVG completo + 27 estados > 50KB pode regredir perf. Mitigação: opção B minimal default.
- **R-3 (baixo):** SVG isométrico shared parece "genérico" vs expectativa de "customização por produto". Mitigação: explicação visual clara no card (label "Dimensões", não "Modelo 3D").
