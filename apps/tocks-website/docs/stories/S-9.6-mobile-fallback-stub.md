# Story S-9.6 — Mobile fallback

## Scope

Viewport <768px NÃO carrega R3F. Detecção via `useMediaQuery` hook SSR-safe (return mobile-default para SSR, hydrate to client true value). `ConfiguratorFallback` organism: hero image grande (reusar `HeroImagePlaceholder` ou Next `<Image>` do produto), grid 3×2 de wood+finish swatches (clicável, estado preview-only — muda texto "Você escolheu: freijó + verniz fosco", NÃO altera imagem em mobile). Lead-gen WhatsApp button igual S-9.4. Design deve feel intencional (editorial), não "mobile-limited excuse".

## Acceptance Criteria (draft)

1. **AC-1** — `useMediaQuery('(max-width: 767px)')` hook criado OR reusado de biblioteca (zero-dep preferido, <20L implementation).
2. **AC-2** — `/colecao/[slug]` em viewport <768px NÃO carrega `Configurator3D` (bundle check: no three.js chunk loaded, inspect devtools Network).
3. **AC-3** — `ConfiguratorFallback` renders: hero image + swatch grid + WhatsApp CTA.
4. **AC-4** — Swatch grid is interactive (visual state selected), updates a `SelectedConfig` text announcement, but não tenta render imagem variante (single hero image shown).
5. **AC-5** — Keyboard nav full; WCAG AA contrast all elements.
6. **AC-6** — Lighthouse `/colecao/[slug]` **mobile** Perf ≥90 (mobile não carrega R3F → perf superior a desktop).

## Notes

- `useMediaQuery` SSR-safe pattern: retorna `false` (desktop) em SSR, client hydrates, componente re-renders. Previne hydration mismatch.
- Design: swatch grid deve honrar Gilded Noir; não UI commodity.
