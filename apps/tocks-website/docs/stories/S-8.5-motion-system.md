# Story 8.5 — Motion System: 5 Classes + GSAP + View Transitions

## Story

**COMO** dev agent do Tocks Website v2
**QUERO** implementar o motion system v2 com 5 classes diferenciadas (hero entrance, product reveal, scroll storytelling, micro-interaction, page transition), consumir Patch #2c (já aplicado em S-8.1), criar hooks `useReveal` + `useChapterScroll`, wire-up GSAP ScrollTrigger nos chapters atelier (S-8.4), implementar Next 16 View Transitions API com directional slides
**PARA** que a elevação Awwwards entregue craft de motion ≥ 9/10 (baseline 5/10 matriz Fase 1), respeitando scarcity-of-motion (Hermès rule), luxury vs gimmick (Tobias), zero scroll-jacking, e Lighthouse Perf não regride vs S-8.1 baseline.

## Context

- **Direção:** B (Gilded Noir Cinemático) motion system — val-head 5 classes differentiated + Patch #2c global (S-8.1 já aplicou).
- **Spec consumível:** `docs/design/elevation-awwwards/06-motion-system-spec.md` (Nova ratificou, cola-direto).
- **Estado atual motion (`globals.css` S-7.3):**
  - 4 keyframes uniformes (`fade-up-hero`, `fade-in-scroll`, `shimmer`, nenhum reduced-motion override) — val-head verdict Sessão 19: "template, não craft".
  - framer-motion removido (S-7.3 Patch #2b) — chunks caíram de ~126KB.
- **Dependência inversa:** S-8.4 já criou chapters com `.chapter-target` class (AC-2 S-8.4 é consumidor). S-8.5 **cria** a class CSS + hook.
- **Next 16 breaking change:** View Transitions via React `<ViewTransition>` component (já lido em `node_modules/next/dist/docs/01-app/02-guides/view-transitions.md`). Requer `experimental: { viewTransition: true }` em `next.config.ts`. Requer React 19.
- **GSAP check:** package.json já tem `gsap` per spec motion 6.3. Validar versão + ScrollTrigger plugin disponível.
- **Art. VII:** hooks em arquivos separados, cada <100 linhas.
- **Princípios não-negociáveis (rams + val-head + tobias-van-schneider):**
  1. Scarcity of motion — max 3 elementos animam simultaneamente
  2. Purposeful delay — delays crescentes, nunca uniformes
  3. Settling, not flicking — ease-out-expo para luxury
  4. Compositor only — apenas `transform` + `opacity`
  5. No scroll-jacking — GSAP pin OK, scroll-speed override NÃO

6. **AC-6 (Classe 4 Micro-interaction):** GIVEN `src/app/globals.css`, WHEN editado, THEN adiciona utility `.micro-interact` com transition 200ms standard cubic-bezier (0.4,0,0.2,1) sobre transform/opacity/color/background-color/border-color. Atualiza seletor `.btn-primary:hover` (micro-settle translate3d(0,-1px,0) + background gold-300) e `:active` (snap back 80ms). Atualiza product-card hover: scale 1.03 (não 1.05) + duration 700ms quartic.

7. **AC-7 (Classe 5 View Transitions):** GIVEN `next.config.ts`, WHEN editado, THEN `experimental: { viewTransition: true }` ativado. GIVEN `src/app/globals.css`, WHEN editado, THEN bloco `::view-transition-old(root)` + `::view-transition-new(root)` + keyframes `fade-out-slight` + `fade-in-up-slight` conforme spec 6.3 Classe 5. GIVEN layout ou Link navigations, WHEN usuário navega (home → colecao → produto), THEN transição suave com expo easing (300ms out / 400ms in) em Chrome. Firefox/Safari graceful degradation (sem animação).

8. **AC-8 (Next 16 docs consulted):** GIVEN `node_modules/next/dist/docs/01-app/02-guides/view-transitions.md`, WHEN lido, THEN @dev decide **approach minimal** para S-8.5:
   - **Approach A (minimal):** apenas CSS `::view-transition-old/new(root)` global — browser-driven, zero React `<ViewTransition>` imports. Funciona em Chrome out-of-box com `experimental.viewTransition: true`.
   - **Approach B (rich):** adicionar `<ViewTransition name="photo-{id}">` em product-card + product-detail para morph shared element.
   - **PM ruling:** @dev **deve** implementar A minimal nesta story. B (morph) fica reservado para Epic 9 (não S-8.5) porque expande escopo >6h.

9. **AC-9 (Prefers-reduced-motion override view transitions):** GIVEN `globals.css`, WHEN editado, THEN adiciona override reduced-motion para view transitions conforme Next 16 docs:
   ```css
   @media (prefers-reduced-motion: reduce) {
     ::view-transition-old(*), ::view-transition-new(*), ::view-transition-group(*) {
       animation-duration: 0s !important;
       animation-delay: 0s !important;
     }
   }
   ```
   Aplicado após Patch #2c de S-8.1.

10. **AC-10 (Scarcity-of-motion guardrail):** GIVEN qualquer viewport na Home, WHEN viewport em primeiro paint, THEN **máximo 3 elementos** animam simultaneamente. Evidência: hero entrance H1 (0ms) + subtitle (250ms) + CTAs (550ms) = **stagger, não simultâneo**. Product cards fora viewport → não animam. Scroll indicator só em 1500ms delay (below fold). Documentar audit em `docs/qa/s-8.5-motion-audit.md`.

11. **AC-11 (Lighthouse Perf não regride):** GIVEN `npm run build && npm start`, WHEN Lighthouse mobile Home + Contato + Produto + Atelier, THEN Perf: Home ≥ 89, Contato ≥ 93, Produto ≥ 90, Atelier ≥ 90 (baseline pós S-8.1). GSAP ScrollTrigger **lazy-load em Atelier page only** (dynamic import) — outras páginas não pagam o chunk.

12. **AC-12 (axe + reduced-motion validado):** GIVEN todas as 5 páginas, WHEN axe-core, THEN 0/0/0/0. GIVEN DevTools emulate `prefers-reduced-motion: reduce`, WHEN todas páginas + navegação, THEN zero animação visível (inclusive view transitions).

13. **AC-13 (Compositor-only check):** GIVEN grep em `globals.css` + novos arquivos, WHEN busca por animações, THEN **zero** animação de `left`, `top`, `width`, `height`, `margin`, `padding`. Apenas `transform`, `opacity`, `filter`, `color`, `background-color`, `border-color`.

14. **AC-14 (Art. VII):** `wc -l` em hooks novos + wrappers: <100 cada. `globals.css` é CSS — exceção Art. VII (mas manter organizado, comentários por seção).

## Dev Notes

### Fontes consumidas

- `docs/design/elevation-awwwards/06-motion-system-spec.md` — spec completa 5 classes + Patch #2c + guardrails
- `node_modules/next/dist/docs/01-app/02-guides/view-transitions.md` — Next 16 API oficial

### Restrições críticas

- **Next 16 breaking:** `experimental.viewTransition: true` + React 19 `<ViewTransition>` component existe MAS @dev faz Approach A (CSS-only, root-level) nesta story. `<ViewTransition>` naming morph fica para Epic 9.
- **Art. VII:** hooks + wrappers <100 linhas cada.
- **Art. IV:** não inventar 6ª classe de motion. Não inventar easings além do spec. Não inventar view transition patterns além do Approach A.
- **Scarcity-of-motion:** max 3 elementos simultâneos (Hermès rule). Documento audit.
- **Compositor-only:** zero `left/top/width/height` animados.

### Val-head ratification (consultation 230aaf35)

- 5 classes differentiated (não uniformes):
  - Hero entrance: ease-out-expo (settling luxury)
  - Product reveal: quartic (produtividade)
  - Scroll storytelling: ease-in-out-cubic (cinematic hold)
  - Micro: standard (quick feedback)
  - Page: expo (smooth exits)
- Delays purposeful não uniformes (0 → 250ms → 550ms hero).
- Product card hover `scale 1.03 + 700ms quartic` (não 1.05 flat 300ms — flicking).

### Rams ratification (consultation bfeaedc)

- Restraint: 5 classes já é o máximo defensável. Sem 6º "attention-grabber" class.
- Zero bounces, zero springs overshoot visíveis (micro-settle 0.01 opcional Chrome 120+ é sutil OK).

### Tobias-van-schneider ratification (Sessão 19 verbatim)

- Luxury vs gimmick rules (spec motion 6.5):
  - Texture animada PROIBIDA — paper-grain static
  - Cursor custom PROIBIDO — Awwwards trend dropou 2024
  - Sem parallax agressivo — scroll é do usuário
  - Motion sugere tempo artesanal, não velocidade SaaS

### Approach A vs B View Transitions (PM ruling)

- **A (CSS-only root):** browser animates entire root, zero React import, funciona com `experimental.viewTransition: true`.
- **B (named morph):** React `<ViewTransition name>` em photo-card + photo-hero para shared-element morph.
- **Ruling:** A nesta story. B em Epic 9 (produto card morph para product detail hero). Separação:
  - A: +0.5h implementação, cobre 80% do "wow" transição.
  - B: +4-6h (naming strategy + testing + Chrome-only validation). Fora de S-8.5.

### Input para S-8.6

- Lighthouse sweep 4 páginas com motion system live — validação final vs thresholds Val-head (spec 6.7).
- Motion audit doc consumido no 5-second test Krug — motion sutil não deve distrair do "atelier de autor" message.

### Riscos identificados

- **R-1 (alto):** GSAP lazy-load pode não funcionar via imports estáticos. Mitigação: `const gsap = (await import('gsap')).default` dentro do hook useEffect OU page-level dynamic import.
- **R-2 (alto):** Next 16 `experimental.viewTransition` pode ter breaking changes não documentadas em patch versions. Mitigação: T-5.1 ler docs + testar em Chrome antes de wire em production routes.
- **R-3 (médio):** Product reveal stagger pode parecer mesmo efeito do hero entrance — val-head differentiation importante. Mitigação: easings distintos (expo hero vs quartic product) + durations diferentes.
- **R-4 (médio):** ScrollTrigger pin em atelier longform pode causar layout shift em mobile mesmo com disable <768px. Mitigação: CLS monitor no Lighthouse.
