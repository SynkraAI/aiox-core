# Story 7.2 — Components + Pages: Molecules, Organisms, HOME/PRODUTO/CONTATO

## Story

**COMO** dev agent do Tocks Website v2
**QUERO** implementar as 5 molecules + 3 organisms novos e refatorar as 3 páginas chave (HOME, colecao/[slug], contato) sobre a fundação de S-7.1
**PARA** que o site refletir as 3 variantes aprovadas pelo squad de design (home.png/produto-berlin.png/contato.png) com feedback WCAG-AA, copy revisada e uso de Server Action do Next.js 16 no formulário concierge.

## Context

- **will-larson (VP Eng)** avaliou se S-7.2 devia ser dividida em 7.2a/7.2b. Decisão PM após consulta: **manter S-7.2 atômica** — molecules/organisms/pages são interdependentes (pages consomem organisms que consomem molecules), dividir geraria PR com molecules órfãs não-mergeáveis. Dentro da story, ordem interna = bottom-up (Bloco 1 molecules → Bloco 2 organisms → Bloco 3 pages).
- **brad-frost** validou hierarquia atomic; lacuna: cada task que cria componente precisa sub-task "validar < 100 linhas" (Art. VII).
- **Next.js 16 breaking changes:** concierge-form usa `<form action={serverAction}>` com `useActionState`. Dev DEVE ler `node_modules/next/dist/docs/` antes de escrever Server Actions.

## Acceptance Criteria

1. **AC-1 (Molecule step-indicator DINÂMICO):** GIVEN componente `molecules/step-indicator.tsx`, WHEN renderizado em página de produto com prop `steps: number`, THEN lê dinamicamente `customizationOptions.length` do produto (2–4 no catálogo real) — NUNCA hardcoded 7. Progress dots + current destacado + ARIA live.

2. **AC-2 (Molecule swatch-picker compound):** GIVEN `<SwatchPicker type="wood">` + `<SwatchPicker.Option value="oak" />`, WHEN usado, THEN padrão Compound Component com Context interno (drasner). Focus ring gold accent full (não `/40`), aria-live "polite", hover:scale-105.

3. **AC-3 (Molecule form-field WCAG):** GIVEN `molecules/form-field.tsx` usado no concierge-form, WHEN focado, THEN border full gold-accent (contraste >= 3:1), label flutuante OU placeholder persistente, `aria-required` e `aria-invalid` suportados, error live region.

4. **AC-4 (Molecule faq-item nativo):** GIVEN `molecules/faq-item.tsx`, WHEN inspecionado, THEN usa `<details>/<summary>` nativo (zero Zustand, zero JS extra). Keyboard nav garantido nativo.

5. **AC-5 (Molecule stats-row):** GIVEN `molecules/stats-row.tsx`, WHEN renderizado, THEN consome `text-pair` de S-7.1, tipografia sans-serif (sem Cormorant gold em números — rams).

6. **AC-6 (Organism image-grid consolidado):** GIVEN `organisms/image-grid.tsx`, WHEN prop `variant` = `project` | `related` (sem variant `client-portrait`), THEN renderiza grid apropriado. Substitui project-showcase + related-products + client-portraits do v1.

7. **AC-7 (Organism cta-block):** GIVEN `organisms/cta-block.tsx`, WHEN renderizado, THEN usa `bg-accent-gold` sólido (SEM `gradient-gold`, removido em S-7.1).

8. **AC-8 (Organism concierge-form Server Action):** GIVEN `organisms/concierge-form.tsx`, WHEN submetido, THEN usa `<form action={submitAction}>` com `useActionState` do Next.js 16 para feedback. `'use client'` APENAS para validação cliente. Action envia dados para endpoint/queue conforme pattern Next 16.

9. **AC-9 (Page HOME):** GIVEN `src/app/page.tsx`, WHEN renderizada, THEN hero left-aligned (NÃO centrado), consome BRAND_COPY.hero, usa `cta-block` e `image-grid variant="project"` (se acervo existir — senão seção mockada + BLOCKED-BY noted).

10. **AC-10 (Page PRODUTO):** GIVEN `src/app/colecao/[slug]/page.tsx`, WHEN acessada com slug válido, THEN renderiza `product-gallery` + `product-specs` + `product-customization` (split em 3 sub-componentes, cada < 100 linhas). `step-indicator` lê `getProductBySlug(slug).customizationOptions.length` dinamicamente.

11. **AC-11 (Page CONTATO):** GIVEN `src/app/contato/page.tsx`, WHEN renderizada, THEN usa `variant="concierge"` em `page-layout.tsx` (NÃO template novo — brad DRY fix), max-width 720px centered, consome `concierge-form` e `faq-item` list.

12. **AC-12 (Art. VII global):** GIVEN qualquer arquivo novo/modificado nesta story, WHEN `wc -l`, THEN < 100 linhas.

13. **AC-13 (Visual match):** GIVEN cada uma das 3 páginas, WHEN screenshot comparado vs `docs/design/approved-variants/{home,produto-berlin,contato}.png`, THEN diff perceptual <= 5% (validação completa fica em S-7.3, mas spot-check ocorre aqui).

### Completion Notes
**Decisoes tecnicas (no order of impact):**

1. **FormField sem `'use client'`:** o atom/molecule eh agnostico. `useId` funciona em Server Components em React 19. Diretiva fica apenas no ConciergeForm pai. Arquivo encolhido de 101→53 linhas apos self-critique Bloco 1.

2. **SwatchPicker unico arquivo compound:** root + Option no mesmo `.tsx` (drasner recomenda isso p/ casos pequenos — Option acessa Context via hook). Atribuicao `SwatchPicker.Option = Option` no final, 96 linhas.

3. **Server Action sem zod (nao instalado):** validacao manual com regex de email + length checks. `redirect()` para WhatsApp com texto pre-populado como fallback ate ter backend de email/CRM. Signature `(_prev, formData)` obrigatoria por `useActionState` (docs/forms.md linhas 195-207).

4. **ConciergeSubmit helper isolado:** `useFormStatus` exige ser filho do `<form>`. Helper de 13 linhas dentro do mesmo arquivo para manter < 100 linhas total (87) e ainda seguir pattern docs/forms.md.

5. **Product-layout split (AC-10):** orchestra `ProductGallery` + `ProductSpecs` + `ProductCustomization` + `ImageGrid` (related). Layout de 132→91 linhas, cada sub < 100 linhas.

6. **HOME 4 product cards editorial:** priorizado `isNew=true` (vertice, rustic) + 2 do resto. Variant `editorial` adiciona chevron top-right fade-in no hover (requisito do prompt).

7. **Footer variant minimal:** logo + copyright + email. Sem endereco fisico (Atelier brand concept, confirmado na story Dev Notes).

8. **CTA home "Transforme seu espaco":** `CtaBlock` com bg solid + border-top gold-deep (AC-7: sem gradient). Consome BRAND_COPY.contact.subtitle.

9. **FAQ ITEMS em contato page:** 5 items (Processo/Prazos/Garantia/Entrega/Valores) direto em array const. Nao extraido para `src/data/` porque sao especificos da page e < 10 items (YAGNI).

**Sem blockers bloqueantes.** Assets Nano Banana 2 continuam como placeholders (ImagePlaceholder com shimmer). Nao impede merge — AC-13 eh spot-check visual, validacao rigorosa de pixel-diff fica em S-7.3.
