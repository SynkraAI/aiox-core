# Story 8.2 — Hero Cinematic: Video Layer + Poster Fallback

## Story

**COMO** dev agent do Tocks Website v2
**QUERO** adicionar um video layer 8-12s (showroom privado slow-dolly, chiaroscuro) ao hero da Home + poster still fallback + novo organism `hero-video.tsx` wrap LCP-safe
**PARA** que a Home entregue o **signature moment** da direção B (Gilded Noir Cinemático) — "a madeira atravessa a luz" — sem degradar LCP mobile, respeitando `prefers-reduced-motion` e Art. VII.

## Context

- **Direção aprovada:** B signature moment = hero video 8-12s loopable (showroom privado, câmera slow-dolly, mesa emergindo da penumbra ao gold accent).
- **Hero atual (`src/components/organisms/hero.tsx` 70L):** CSS-only, texture radial gold 3% wallpaper genérico. MANTIDO em S-7.3 como baseline LCP-safe. Agora recebe **layer de video** por cima (vídeo é ambiente, não substitui LCP text).
- **Spec shot list** (master-plan Fase 3):
  - 0-2s: penumbra, zoom lento 35mm em detalhe textural madeira
  - 2-5s: dolly-out mesa emerge chiaroscuro, luz quente 3200K pincela borda
  - 5-8s: hold em composição 1/3, gold accent em ferragem
  - 8-12s: fade crossdissolve loop seamless
- **Constraints técnicos:**
  - Desktop: 1920x1080, H.264 + WebM VP9 + HEVC fallback, max 2MB
  - Mobile: 720x1280 vertical, max 1.5MB (ou hide video, mostrar só poster)
  - Poster frame: JPG/AVIF < 80KB, primeiro frame do video
  - Autoplay muted, loop, playsinline, sem áudio
- **BLOCKED-BY asset:** Nano Banana 2 quota esgotada — video real ainda não existe. S-8.2 entrega **infra técnica** + **poster placeholder**; video pode ser hot-swapped quando asset chegar. **Fallback accept:** poster still estático passa Lighthouse sozinho. Video é enhancement progressivo.
- **Art. VII:** hero-video.tsx <100 linhas. hero.tsx pode integrar via slot/child.
- **Next 16 guardrail:** ler `node_modules/next/dist/docs/` sobre `next/image` para poster (AVIF auto-optimize) antes de implementar.

## Acceptance Criteria

1. **AC-1 (Organism hero-video):** GIVEN `src/components/organisms/hero-video.tsx` (novo), WHEN renderizado com props `{ videoSrc?: string, posterSrc: string, alt: string }`, THEN retorna `<video>` com `autoPlay muted loop playsInline preload="metadata"` + `<source>` H.264 + WebM + HEVC + fallback `<img>` poster se video falha. Arquivo <100 linhas.

2. **AC-2 (Poster LCP primary):** GIVEN Home primeiro paint, WHEN Lighthouse mobile prod mede LCP, THEN LCP **é o poster still** (não o video). Target LCP ≤ 2.5s mobile. Poster servido via `next/image` com `priority` + AVIF + `fetchPriority="high"`.

3. **AC-3 (Video enhancement progressivo):** GIVEN conexão rápida + autoplay habilitado, WHEN video carrega, THEN substitui poster sem flash (`object-fit: cover`, `opacity` transition 400ms). GIVEN conexão lenta OU `prefers-reduced-motion: reduce` OU `<video>` erro, WHEN hero renderiza, THEN **poster só** (video hidden ou never inserted).

4. **AC-4 (Integração hero.tsx):** GIVEN `src/components/organisms/hero.tsx`, WHEN atualizado, THEN consome `<HeroVideo>` como background layer (`absolute inset-0`, behind text), substituindo o radial-dot texture atual. Texto H1/subtitle/CTAs permanecem na frente (`relative z-10`). Hero atual mantém <100 linhas (Art. VII).

5. **AC-5 (Shadow cinematic):** GIVEN `<HeroVideo>` container, WHEN renderizado, THEN aplica `var(--shadow-cinematic)` + `var(--shadow-gold-inset)` conforme token S-8.1. Gradient overlay noir `from-[var(--noir-deep)] via-transparent to-[var(--noir-deep)]` preserva legibilidade do H1.

6. **AC-6 (Heading display Fraunces):** GIVEN H1 hero, WHEN inspecionado, THEN usa class `.heading-display` (Fraunces opsz 144, weight 300, tracking negative -0.03em) conforme token S-8.1. Substitui o `Heading as="h1"` atual — opcional ou additive (decisão @dev: preservar Heading atom + adicionar `.heading-display` class via prop `className`).

7. **AC-7 (prefers-reduced-motion):** GIVEN user prefere reduced motion, WHEN Home carrega, THEN `<video>` **não é montado** (not just paused — literalmente ausente do DOM ou `display: none` via CSS). Fallback: poster still full-frame. Validar via DevTools emulate.

8. **AC-8 (Accessibility):** GIVEN video element, WHEN inspecionado, THEN `aria-hidden="true"` (é decorativo — alt visual está no poster `<img alt>`). Video mudo sem áudio. axe-core 0/0/0/0.

9. **AC-9 (Mobile):** GIVEN viewport <768px, WHEN hero renderiza, THEN video **é opcional**: implementar lógica via `<source media="(min-width: 768px)">` ou omitir video mobile inteiramente (decisão @dev baseada em peso do asset). Mobile **deve** passar Lighthouse Perf ≥ 90.

10. **AC-10 (Asset placeholder entrega):** GIVEN video real ainda não existe (Nano Banana 2 quota), WHEN S-8.2 entrega, THEN:
    - `public/videos/hero-showroom.mp4` = placeholder 8s preto + texto "VIDEO PLACEHOLDER" (ou ausente — `videoSrc={undefined}`)
    - `public/images/hero-poster.avif` = poster placeholder noir+gold com texto "HERO POSTER PLACEHOLDER" em Fraunces
    - README `public/videos/README.md` documenta swap path quando asset real chegar
    - Sem isso, AC-2 não é testável; esta é entrega mínima viável.

11. **AC-11 (Build + perf):** GIVEN `npm run build && npm start`, WHEN Lighthouse mobile Home, THEN Perf ≥ 89 (não regride S-8.1 baseline) · LCP ≤ 2.5s · CLS ≤ 0.05.

12. **AC-12 (Art. VII):** `wc -l src/components/organisms/hero-video.tsx` < 100 linhas. `wc -l src/components/organisms/hero.tsx` < 100 linhas.

### Decisões técnicas (Karpathy — explicitadas)

1. **`useSyncExternalStore` > `useEffect+setState`** — eslint (react-hooks/set-state-in-effect) rejeita `setState` síncrono em effect. `useSyncExternalStore` é o pattern React 19 idiomatic para "read external reactive value". SSR default = `true` (reduced) → não renderiza `<video>` no HTML, elimina flash no client de reduced-motion.
2. **JPG placeholder, não AVIF** — `next.config.ts` já tem `images.formats: ['image/avif','image/webp']`, então `next/image` converte automaticamente. Poster real (quando chegar) pode ser JPG ou AVIF — hot-swap é drop-in.
3. **3 codec sources em vez de 2** — interface expõe `videoSrc` (H.264) + `videoSrcWebm` (VP9) + `videoSrcHevc` (H.265). Ordem `<source>`: WebM → HEVC → H.264 fallback. Safe com spec (Art. IV — só os 3 codecs aprovados).
4. **`videoSrc={undefined}` por default** — sem asset real, HeroVideo renderiza só poster. AC-3 fallback path validado.
5. **Next 16 `priority` deprecated** — usei `preload fetchPriority="high"` (consulta `node_modules/next/dist/docs/.../image.md` confirmou deprecation).
6. **`pointer-events-none` no overlay gradient** — garante CTAs clicáveis sobre o overlay (regressão sutil se omitido).
