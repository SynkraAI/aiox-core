## Acceptance Criteria

1. **AC-1 (WCAG AA global):** GIVEN axe-core rodado em `/`, `/colecao/tenro-luxo`, `/contato`, WHEN relatório gerado, THEN ZERO violações sério/crítico. Warnings moderate documentados com plano de fix ou waiver justificado.

2. **AC-2 (Focus states):** GIVEN cada elemento interativo (links, buttons, swatch-picker options, form-fields), WHEN recebe focus via Tab, THEN focus ring visível com contraste >= 3:1 vs background, offset 2px conforme token de S-7.1.

3. **AC-3 (Keyboard nav):** GIVEN o site inteiro, WHEN navegado APENAS com teclado (Tab/Shift+Tab/Enter/Space/Esc/Arrow), THEN: (a) `<details>` FAQ abre/fecha, (b) swatch-picker navegável com Arrow keys + Enter, (c) concierge-form submetível, (d) NENHUM focus trap ou elemento inacessível.

4. **AC-4 (Lighthouse budget):** GIVEN Lighthouse CI rodado em mobile + desktop para as 3 páginas, WHEN scores medidos, THEN Performance >= 90, Accessibility >= 95, Best Practices >= 95, SEO >= 95. Scores abaixo DEVEM ter Dev Notes explicando causa.

5. **AC-5 (Visual regression):** GIVEN screenshot de cada uma das 3 páginas (viewport 1440x900), WHEN pixelmatch vs `docs/design/approved-variants/{home,produto-berlin,contato}.png`, THEN diff perceptual <= 5%. Diffs maiores documentados com screenshot + justificativa (ex.: Berlin referência, produto real é Tenro Luxo).

6. **AC-6 (Copy review pt-BR):** GIVEN cada página renderizada, WHEN copy inspecionada, THEN: (a) ZERO ponto de exclamação, (b) presença de "Atelier", "Peça", "Sob medida", "Projetos Realizados", "Dúvidas Frequentes", (c) ZERO "Nossas Peças na Coração", "Considerações do Atelier", "visita ao atelier" (endereço), (d) sem dados inventados (telefone/email batem com constants.ts).

7. **AC-7 (Server Action smoke):** GIVEN concierge-form na `/contato`, WHEN submetido com dados válidos, THEN Server Action executa e `useActionState` retorna success state visível; WHEN submetido com campo obrigatório vazio, THEN validação cliente bloqueia + error live region lê mensagem.

8. **AC-8 (Unit tests componente-chave):** GIVEN componentes críticos desta epic (text-pair, step-indicator, swatch-picker, form-field, faq-item, concierge-form), WHEN suite rodada, THEN cobertura >= 80% em cada, todos passam.

9. **AC-9 (Art. VII audit):** GIVEN `git diff` da epic inteira (S-7.1+S-7.2), WHEN `wc -l` em cada arquivo novo/modificado, THEN 100% dos componentes < 100 linhas. Violações são BLOQUEANTES.

10. **AC-10 (Brand/assets blocker log):** GIVEN `image-grid variant="project"` em HOME, WHEN inspecionado, THEN se ainda usa placeholders, isso está documentado em `docs/qa/s-7.3-known-gaps.md` como BLOCKED-BY Nano Banana 2 — NÃO bloqueia merge desta story (é expectativa conhecida).
