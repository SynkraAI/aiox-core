# Story S-9.1 — View-Transitions Approach B Foundation

## Scope

Estabelecer a infraestrutura para morph cross-route: React `<ViewTransition>` wrappers com `name={`product-${slug}`}` em product-card AND product page hero; header anchoring via `viewTransitionName: 'site-header'` CSS-only; reduced-motion cascade estendido para `::view-transition-*(*)` em `globals.css`. Sem animação custom ainda (default browser morph) — S-9.2 polish. Unblocks todo o Theme A chain.

## Acceptance Criteria (draft)

1. **AC-1** — `<ViewTransition>` imported from `'react'` (React 19 stable) no product-card e em product page hero (`ProductLayout`); both wrap `<ImagePlaceholder>` / `next/image` com `name={`product-${product.slug}`}`.
2. **AC-2** — Header anchored: `style={{ viewTransitionName: 'site-header' }}` em `<header>`; CSS em `globals.css` bloco `::view-transition-group(site-header) { animation: none; z-index: 100; }` + `::view-transition-old(site-header) { display: none; }`.
3. **AC-3** — Reduced-motion extension: bloco `@media (prefers-reduced-motion: reduce)` em `globals.css` cobre `::view-transition-old(*)`, `::view-transition-new(*)`, `::view-transition-group(*)` com `animation-duration: 0s !important`.
4. **AC-4** — Lighthouse home `/` Perf ≥85 (no regression vs Epic 8 closure), `/colecao` ≥90, `/colecao/[slug]` ≥90 (3D ainda não montado, pure foundation).
5. **AC-5** — Smoke test: navigate home → product → back in Chrome + Safari + Firefox + reduced-motion emulation — zero errors console, morph visibly works Chrome/Edge, gracefully no-op elsewhere.
