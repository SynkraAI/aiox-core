# Recon Engine — Skill Reconnaissance

Analyze any skill to build a complete profile for stress testing.

## Input

- `skill_name`: name of the skill to analyze (e.g., "forge", "lp-generator")

## Output

- `skill-profile.yaml` written to `.stress-test/` in the fixture directory

## Algorithm

### Step 1: Locate the Skill

Search in this order:
1. `~/aios-core/skills/{skill_name}/SKILL.md`
2. `~/aios-core/squads/{skill_name}/README.md`
3. Glob `~/aios-core/skills/*/SKILL.md` and grep for skill_name in description
4. If not found: ABORT with error "Skill '{skill_name}' nao encontrada no ecossistema."

### Step 2: Parse SKILL.md

Read the entire SKILL.md and extract:

#### 2a. Frontmatter (YAML block between `---`)
- `name` — canonical name
- `description` — what it does
- `allowed-tools` — which tools the skill uses (Read, Write, Bash, Agent, etc.)
- `argument-hint` — accepted commands/args
- `version` — current version
- `category` — skill category
- `tags` — search tags
- `lazy_load` — whether it loads references on demand
- `context_budget` — token budget

#### 2b. Skill Type Classification

Determine type based on content analysis:

| Type | Signals |
|------|---------|
| `pipeline` | Has numbered phases, phase files in `phases/` or `engine/`, progress tracking |
| `command-based` | Has command routing table, multiple entry points, state machine |
| `single-flow` | One input → one output, no phases, no state persistence |
| `orchestrator` | Routes to other skills/agents, has delegation patterns |
| `generator` | Creates files/artifacts as primary output (code, content, images) |

#### 2c. Entry Points

Extract all ways to invoke the skill:
- From `argument-hint` in frontmatter
- From command routing tables in body
- From discovery questions (each answer path = entry point)
- From subcommands mentioned in headings

#### 2d. Discovery Questions

Extract the discovery questions section:
- How many questions
- What options each has
- Whether they're mandatory or skippable

#### 2e. Artifacts Produced

Identify all files the skill creates:
- Output files mentioned in instructions
- State files (`.aios/`, `progress.yaml`, `state.json`, etc.)
- Reports or generated content
- Temporary files

#### 2f. Dependencies

Identify external dependencies:
- **Agents**: `@dev`, `@qa`, `@analyst`, etc. (grep for `@` mentions)
- **Other skills**: `/skill-name` invocations or skill references
- **MCP servers**: playwright, exa, etc.
- **Scripts**: files in `scripts/` directory
- **External tools**: npm, git, node, python, etc.
- **Engine modules**: files in `engine/` loaded during execution

#### 2g. State Files

Identify what state the skill reads and writes:
- `.aios/` directory patterns
- YAML/JSON state files
- Progress tracking files
- Session/history files

### Step 3: Risk Analysis

Based on the profile, identify potential stress points:

| Risk Category | What to Look For |
|---------------|-----------------|
| `missing-guard` | File reads without existence checks |
| `state-corruption` | State files without validation/recovery |
| `agent-dependency` | Hard dependency on specific agents |
| `mcp-dependency` | Features that require MCP servers |
| `context-overflow` | Large files loaded without lazy_load |
| `no-resume` | No mechanism to continue after interruption |
| `no-idempotency` | Running twice produces different/broken results |
| `hardcoded-paths` | Absolute paths that won't work in fixture |
| `no-error-handling` | Missing try/catch or error recovery |
| `runtime-specific` | Features that only work in Claude Code (Agent tool, Skill tool) |

### Step 4: Generate Profile

Write `skill-profile.yaml`:

```yaml
# Auto-generated by skill-stress-test recon
# Do not edit manually

target_skill: {name}
skill_path: {absolute path to SKILL.md}
skill_type: {pipeline|command-based|single-flow|orchestrator|generator}
version: {version}

entry_points:
  - command: "{command}"
    description: "{what it does}"
  - command: "{command}"
    description: "{what it does}"

discovery_questions:
  count: {N}
  mandatory: {true|false}
  questions:
    - "{question text}"

artifacts_produced:
  - path: "{relative path}"
    type: {output|state|report|temp}
  - path: "{relative path}"
    type: {output|state|report|temp}

dependencies:
  agents: [{list}]
  skills: [{list}]
  mcp_servers: [{list}]
  scripts: [{list}]
  external_tools: [{list}]
  engine_modules: [{list}]

state_files:
  reads: [{list of paths}]
  writes: [{list of paths}]

risk_areas:
  - category: "{risk category}"
    description: "{what could go wrong}"
    severity: {low|medium|high|critical}
    affected_module: "{file path}"

recommended_fixture: {node-minimal|python-minimal|brownfield|chaos-empty}
recommended_depth: {quick|full}
estimated_scenarios:
  quick: {N}
  full: {N}
```

### Step 5: Summary Display

Show the profile summary to the user:

```
=== RECON COMPLETO ===
Skill: {name} v{version} ({type})
Path: {skill_path}

Entry points: {count}
  {list first 5, "... e mais {N}" if > 5}

Dependencias: {agents + skills + mcp count}
  {list all}

Riscos identificados: {count}
  {emoji per severity} {description} ({affected_module})

Fixture recomendado: {archetype}
Cenarios estimados: {quick} (rapido) / {full} (completo)
```
