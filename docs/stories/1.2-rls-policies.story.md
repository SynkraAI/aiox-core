# Story 1.2: Implement RLS Policies (Multi-tenant Security)

## Story Statement

Implement Row-Level Security (RLS) policies on all database tables to enforce multi-tenant isolation and role-based access. **This is critical for data security** — ensures users can only see their own data or data they have permission to access.

## Core RLS Policies

| Table | Policy | Rule |
|-------|--------|------|
| `users` | "Users view self or admin" | View self OR if ADMIN |
| `customers` | "All authenticated see" | Anyone authenticated |
| `sales` | "By role" | ADMIN/FINANCEIRO see all; COMERCIAL see own (seller_id) |
| `commissions` | "By role" | Same as sales |
| `accounts` | "FINANCEIRO only" | Only ADMIN/FINANCEIRO |
| `movements` | "FINANCEIRO only" | Only ADMIN/FINANCEIRO |
| `audit_log` | "ADMIN only" | Only ADMIN |

## Acceptance Criteria

- [x] RLS enabled globally on all 8 tables
- [x] Each policy tested (cross-tenant leak = FAIL)
- [x] Performance verified (queries <100ms on 1M rows)
- [x] Admin can see everything (policy allows)
- [x] Comercial cannot see other seller's data
