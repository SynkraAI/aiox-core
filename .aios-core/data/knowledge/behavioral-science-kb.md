## 1. Fogg Behavior Model

### 1.1 B = MAP

```
Behavior = Motivation × Ability × Prompt

For a behavior to occur, three elements must converge at the same moment:

┌──────────────────────────────────────────┐
│                                          │
│  High │     ●                            │
│       │    ● ● ●   BEHAVIOR HAPPENS      │
│  M    │   ● ● ● ●  (above the curve)     │
│  O    │  ● ● ● ● ●                       │
│  T    │ ──────────── Action Line ──────── │
│  I    │                                   │
│  V    │   BEHAVIOR DOESN'T HAPPEN         │
│  A    │    (below the curve)              │
│  T    │                                   │
│  I    │                                   │
│  O    │                                   │
│  N    │                                   │
│  Low  │                                   │
│       └───────────────────────────────────│
│         Hard ◄── ABILITY ──► Easy         │
│                                          │
│  + PROMPT must be present                │
└──────────────────────────────────────────┘

Key Insight:
If motivation is HIGH, ability can be LOW (and vice versa).
But the PROMPT must always be present.
No prompt = no behavior, regardless of motivation and ability.
```

### 1.2 Motivation Sources (Fogg)

```
Three Core Motivators:

1. SENSATION
   ├── Pleasure (seek) ↔ Pain (avoid)
   └── Example: Chocolate is pleasurable, exercise pain is avoided

2. ANTICIPATION
   ├── Hope (seek) ↔ Fear (avoid)
   └── Example: Hope for promotion drives work; fear of failure drives study

3. SOCIAL COHESION
   ├── Acceptance (seek) ↔ Rejection (avoid)
   └── Example: Social media likes (acceptance); fear of being left out (rejection)

Motivation Waves:
Motivation naturally fluctuates. Don't design for peak motivation.
Design for when motivation is at its LOWEST.

"Motivation is unreliable. Design for lazy, unmotivated humans."
```

### 1.3 Ability Factors (Simplicity Chain)

```
Ability is determined by the SCARCEST resource:

1. TIME - How long does it take?
   ├── Less time = easier = more likely
   └── Example: 1-click checkout > 5-page checkout

2. MONEY - How much does it cost?
   ├── Less cost = easier = more likely
   └── Example: Free trial > paid signup

3. PHYSICAL EFFORT - How much exertion?
   ├── Less effort = easier = more likely
   └── Example: Voice command > typing

4. BRAIN CYCLES - How much thinking?
   ├── Less thinking = easier = more likely
   └── Example: "Buy again" > "Search and compare"

5. SOCIAL DEVIANCE - How much does it violate norms?
   ├── More socially acceptable = easier
   └── Example: Recycling (normal) > composting (unusual)

6. NON-ROUTINE - How much does it differ from habit?
   ├── More routine = easier = more likely
   └── Example: Adding to existing commute > creating new trip

The bottleneck is always the scarcest factor.
Find it. Fix it. Watch behavior change.
```

### 1.4 Prompt Types

```
Three Types of Prompts:

1. FACILITATOR (high motivation, low ability)
   ├── Makes the behavior easier to do
   ├── "Click here to auto-fill your address"
   └── Trigger + simplification

2. SPARK (low motivation, high ability)
   ├── Increases motivation to act
   ├── "Your friend just joined! Start a conversation"
   └── Trigger + emotional appeal

3. SIGNAL (high motivation, high ability)
   ├── Simple reminder that it's time
   ├── Push notification: "Time for your daily check-in"
   └── Trigger only (person is ready)

Prompt Placement:
├── In the moment of readiness (not before, not after)
├── In the user's natural flow (not disruptive)
├── Connected to an existing routine (anchor)
└── Clear and specific (what to do, not vague)
```

## 2. Tiny Habits Methodology

### 2.1 The Tiny Habits Recipe

```
After I [ANCHOR MOMENT], I will [TINY BEHAVIOR].

Components:
├── ANCHOR: An existing routine that happens reliably
│   Examples: After I pour my coffee, After I sit at my desk,
│   After I open my laptop, After I brush my teeth
│
├── TINY BEHAVIOR: The smallest version of the desired behavior
│   Examples: Do 2 pushups, Write one sentence, Read one page,
│   Drink one sip of water, Take one deep breath
│
└── CELEBRATION: Immediate positive emotion after the behavior
    Examples: Say "I'm awesome!", Do a fist pump,
    Smile at yourself, Hum a victory tune

The Recipe in Practice:
"After I pour my morning coffee, I will open my journal and write one sentence."
*Celebrates* "Yes! I'm a writer!"

Why it works:
1. Anchoring to existing routine = reliable prompt
2. Making it tiny = ability is maxed out
3. Celebration = creates positive emotion = wires the habit
```

### 2.2 Scaling Up Tiny Habits

```
Growth Phases:

Phase 1: TINY (Days 1-7)
├── Do the absolute minimum
├── 2 pushups, not 20
├── 1 sentence, not 1 page
├── Focus on DOING IT, not doing it well
└── Celebrate every time

Phase 2: NATURAL GROWTH (Days 8-30)
├── Behavior naturally expands
├── 2 pushups become 5 (because you want to)
├── 1 sentence becomes a paragraph
├── Don't force expansion, let it happen
└── If you resist, you've grown too fast - scale back

Phase 3: HABIT (Days 30+)
├── Behavior feels automatic
├── Missing it feels wrong
├── Expand deliberately now
├── Add complementary habits
└── Stack new tiny habits on this one

Key Rule: NEVER increase the minimum requirement.
The minimum stays tiny forever.
Some days you'll do more. Some days you'll do the minimum.
Both are perfect.
```

### 2.3 Behavior Design for Teams/Products

```
Product Application of Tiny Habits:

Onboarding:
├── Anchor: After signing up
├── Tiny Behavior: Complete ONE micro-task (not a full tutorial)
├── Celebration: Confetti animation, progress badge
└── Example: Slack → "Send your first message in #general"

Engagement:
├── Anchor: After opening the app
├── Tiny Behavior: View one piece of personalized content
├── Celebration: Social feedback, streak counter
└── Example: Duolingo → Complete one 2-minute lesson

Retention:
├── Anchor: At specific time (notification)
├── Tiny Behavior: One small action in the product
├── Celebration: Progress visualization, achievement
└── Example: Headspace → "1 minute meditation available"
```

## 3. Hooked Model

### 3.1 Nir Eyal's Hook Cycle

```
The Hook: A 4-phase cycle that creates habits

┌──────────────────────────────────────────┐
│                                          │
│     1. TRIGGER ──────► 2. ACTION         │
│         ▲                   │            │
│         │                   ▼            │
│     4. INVESTMENT ◄─── 3. VARIABLE       │
│                         REWARD           │
│                                          │
└──────────────────────────────────────────┘

The more cycles completed, the stronger the habit.
Goal: Move from external triggers to internal triggers.
```

### 3.2 Phase Details

#### Phase 1: Trigger

```
External Triggers (early stage):
├── Paid: Ads, sponsored content
├── Earned: Press mentions, viral content, SEO
├── Relationship: Word of mouth, referrals
└── Owned: Email, push notifications, app icon

Internal Triggers (habit stage):
├── Emotions: Boredom → Instagram, Loneliness → Facebook
├── Situations: Waiting → Twitter, Cooking → YouTube
├── Routines: Morning → Email, Evening → Netflix
└── Pain points: Uncertainty → Google

Goal: External triggers should install internal triggers.
After enough cycles, the user triggers themselves.

"What internal trigger does your product address?"
If you can't answer this, you don't have a habit-forming product.
```

#### Phase 2: Action

```
Action = The simplest behavior in anticipation of reward

Fogg Behavior Model applies:
├── Make the action as SIMPLE as possible
├── Motivation comes from anticipation of reward
└── Trigger (Phase 1) serves as the prompt

Examples of Minimal Actions:
├── Instagram: Open app → Scroll (one thumb movement)
├── Google: Type query → Press enter
├── Twitter: Open app → Read first tweet
├── TikTok: Open app → Watch video (auto-plays)
├── Slack: See notification → Click to read

Key: Reduce friction to near-zero for the initial action.
The action should take < 3 seconds.
```

#### Phase 3: Variable Reward

```
Three Types of Variable Rewards:

1. REWARDS OF THE TRIBE (Social)
   ├── Likes, comments, followers
   ├── Recognition, status
   ├── Cooperation, competition
   └── Examples: Social media engagement, Stack Overflow reputation

2. REWARDS OF THE HUNT (Resources)
   ├── Material goods, money, information
   ├── The thrill of the chase/search
   ├── Scrolling for the next great post
   └── Examples: Pinterest scrolling, slot machines, sales deals

3. REWARDS OF THE SELF (Mastery)
   ├── Personal achievement, competency
   ├── Completion, consistency
   ├── Self-improvement
   └── Examples: Video game levels, Duolingo streaks, Fitbit goals

Why VARIABLE is crucial:
├── Predictable rewards become boring (hedonic adaptation)
├── Variable rewards maintain dopamine response
├── The ANTICIPATION of reward is more powerful than the reward itself
├── Intermittent reinforcement creates strongest habits
└── Like a slot machine: not knowing when the reward comes = engagement

"The unknown is fascinating." - B.F. Skinner
```

#### Phase 4: Investment

```
Investment = User puts something into the product that:
├── Increases the value of the next cycle
├── Creates switching costs
├── Stores value for future use
└── Loads the next trigger

Types of Investment:
├── Data: Profile info, preferences, history
├── Content: Posts, photos, reviews, messages
├── Followers: Social connections, network
├── Reputation: Ratings, badges, level
├── Skill: Learning the interface, keyboard shortcuts
└── Money: Subscriptions, in-app purchases

Examples:
├── Pinterest: Pinning (curated boards) → Better recommendations next time
├── Tinder: Swiping (preferences) → Better matches next time
├── LinkedIn: Adding connections → More relevant feed next time
├── Spotify: Creating playlists → More personalized Discover Weekly

Key: Investment should happen AFTER the reward.
User is most willing to invest when they've just been rewarded.
```

### 3.3 The Morality of Manipulation

```
Nir Eyal's Manipulation Matrix:

                    Does the maker use the product?
                    YES                 NO
              ┌──────────────────┬──────────────────┐
Materially    │                  │                  │
improves      │   FACILITATOR    │   PEDDLER        │
user's life   │   (Ethical)      │   (Questionable) │
  YES         │   Build this!    │   Use caution    │
              ├──────────────────┼──────────────────┤
              │                  │                  │
  NO          │   ENTERTAINER    │   DEALER         │
              │   (Fun, but be   │   (Exploitative) │
              │   careful)       │   Don't build    │
              └──────────────────┴──────────────────┘

Ethics Test:
1. Would I use this product?
2. Does it genuinely help the user?
3. Am I comfortable if my use tactics were public?
4. Does the user maintain autonomy?
5. Can the user easily disengage?
```

## 4. System 1 and System 2

### 4.1 Daniel Kahneman's Dual Process Theory

```
SYSTEM 1 (Fast Thinking):
├── Automatic, effortless
├── Always on, can't be turned off
├── Emotion-driven
├── Pattern matching
├── Intuitive judgments
├── Handles 95% of daily decisions
├── Prone to cognitive biases
└── Examples: Reading text, driving familiar route, recognizing faces

SYSTEM 2 (Slow Thinking):
├── Deliberate, effortful
├── Requires attention and concentration
├── Logic and reason-driven
├── Step-by-step analysis
├── Complex calculations
├── Handles 5% of decisions (important ones)
├── Lazy - only engages when necessary
└── Examples: Solving math, learning new skill, complex decisions

Key Insight for Product Design:
Most user interactions happen in System 1.
Design for System 1 (fast, intuitive, minimal thinking).
Only require System 2 for critical decisions (purchases, data entry).
```

### 4.2 Prospect Theory

```
Key Findings:

1. LOSS AVERSION
   Losses are felt ~2.5x more intensely than equivalent gains.
   Losing $100 feels like losing $250 of potential gain.

   Application:
   ├── Frame as avoiding loss, not gaining benefit
   ├── "Don't miss out" > "You could benefit"
   ├── Free trials exploit loss aversion (take it away → they buy)
   └── "You'll lose access to [features] if you downgrade"

2. REFERENCE DEPENDENCE
   People evaluate outcomes relative to a reference point.

   Application:
   ├── Set the reference point in your favor (anchoring)
   ├── "Was $199, now $99" (reference = $199)
   ├── Show premium plan first (reference = expensive)
   └── Before/after comparisons

3. DIMINISHING SENSITIVITY
   The difference between $0 and $100 feels bigger than
   the difference between $1,000 and $1,100.

   Application:
   ├── Frame in small amounts: "$3/day" not "$90/month"
   ├── Percentage framing changes with magnitude
   │   └── "Save 50%" works for small amounts, "$500 off" for large
   └── Bundling: Add cheap items to expensive purchase

4. CERTAINTY EFFECT
   People overweight outcomes that are certain vs probable.

   Application:
   ├── "Guaranteed results" > "90% chance of results"
   ├── Money-back guarantees reduce perceived risk to zero
   └── "Free shipping" (certain saving) > "Might save $X"
```

## 5. Nudge Theory

### 5.1 Thaler & Sunstein's Framework

```
A NUDGE is:
├── Any aspect of the choice architecture that alters behavior
├── In a predictable way
├── WITHOUT forbidding options
├── WITHOUT significantly changing economic incentives
└── Easy and cheap to avoid

NOT a Nudge:
├── Mandates or bans
├── Financial penalties or rewards
├── Removing options
└── Deception

Libertarian Paternalism:
"We guide choices toward outcomes that are better for people,
while preserving their freedom to choose otherwise."
```

### 5.2 Types of Nudges

```
1. DEFAULT OPTIONS (Most powerful nudge)
   ├── People tend to stick with defaults
   ├── Example: Organ donation opt-out vs opt-in
   │   └── Opt-out countries: ~90% donors
   │   └── Opt-in countries: ~15% donors
   ├── Product example: Annual billing pre-selected
   └── Power: 3-10x more effective than other nudges

2. SOCIAL NORMS
   ├── "9 out of 10 people in your area recycle"
   ├── "Most guests reuse their towels"
   ├── Product: "Join 50,000 professionals who..."
   └── Power: Works when behavior is uncertain

3. SIMPLIFICATION
   ├── Reduce complexity of forms and processes
   ├── Pre-fill information where possible
   ├── Reduce number of options (choice overload)
   └── Power: Removes friction barriers

4. SALIENCE
   ├── Make important information visible
   ├── Calorie counts on menus
   ├── Energy usage comparisons
   └── Power: Triggers awareness

5. FEEDBACK
   ├── Show progress and results
   ├── Real-time energy monitor in homes
   ├── Step counters, spending trackers
   └── Power: Creates awareness loop

6. COMMITMENT DEVICES
   ├── Ask people to commit to future behavior
   ├── "Save More Tomorrow" (auto-escalating savings)
   ├── Product: Annual commitment with monthly benefit
   └── Power: Leverages present-bias

7. PRIMING
   ├── Subtle cues that influence subsequent behavior
   ├── Images of eyes near honesty boxes → more payment
   ├── Clean scents → tidier behavior
   └── Power: Unconscious influence
```

### 5.3 EAST Framework (Behavioral Insights Team)

```
Make it:

E - EASY
├── Set smart defaults
├── Reduce friction (fewer steps, pre-fill)
├── Simplify messages (clear, jargon-free)
└── "If something is easy, people will do more of it"

A - ATTRACTIVE
├── Make key information salient (color, size, position)
├── Personalize communications
├── Use incentives wisely
└── "Attention is scarce. Design to capture it."

S - SOCIAL
├── Show what others are doing
├── Use the power of networks
├── Encourage commitments to others
└── "We are profoundly influenced by what others do"

T - TIMELY
├── Prompt when people are most receptive
├── Consider immediate costs and benefits
├── Help people plan their response
└── "The same message at the right time > the perfect message at the wrong time"
```

## 6. Choice Architecture

### 6.1 Principles of Choice Architecture

```
1. NUMBER OF OPTIONS
   ├── Too few: Feels restrictive
   ├── Too many: Choice paralysis (Paradox of Choice)
   ├── Sweet spot: 3-5 options for most decisions
   ├── Famous study: Jam experiment (Iyengar & Lepper)
   │   └── 24 jams → 3% purchased
   │   └── 6 jams → 30% purchased
   └── Product: 3 pricing tiers, not 7

2. ORDER AND POSITIONING
   ├── Primacy effect: First option remembered best
   ├── Recency effect: Last option remembered second-best
   ├── Middle bias: People tend to choose middle option
   ├── Application: Put your preferred option in the middle
   └── Menu psychology: Most profitable items top-right

3. PARTITIONING
   ├── Breaking categories affects perception
   ├── Separate bad news, combine good news
   ├── Bundle fees into one price (vs itemized)
   └── Or itemize value to show what's included

4. DECOY EFFECT
   ├── Add an asymmetrically dominated option
   ├── Classic: Small $3, Large $7, Medium $6.50
   │   └── Medium makes Large look like better deal
   ├── The Economist: Web-only $59, Print $125, Web+Print $125
   │   └── Print-only makes bundle look like a steal
   └── Use to steer toward your preferred option

5. FRAMING
   ├── Same information, different presentation
   ├── "90% fat-free" > "10% fat" (same yogurt)
   ├── "Save $50" vs "Save 33%" (depends on price point)
   ├── Loss frame > Gain frame (for action)
   └── Positive frame > Negative frame (for adoption)
```

### 6.2 Choice Architecture for Products

```
Pricing Page Architecture:

Tier 1: FREE / BASIC (Anchor low, drive adoption)
├── Limited features
├── Purpose: Build habit, create investment
└── Nudge toward paid: Feature walls, usage limits

Tier 2: PRO / RECOMMENDED (Your target tier)
├── "Most Popular" badge (social proof nudge)
├── Highlighted with different color/border
├── Best value positioning
├── Annual pricing shown with monthly equivalent
└── This is where 60-70% of revenue should come from

Tier 3: ENTERPRISE / PREMIUM (Anchor high)
├── Makes Tier 2 look affordable by comparison
├── Custom pricing ("Contact us")
├── Premium features most users don't need
└── Serves as price anchor

Toggle: Monthly / Annual
├── Annual pre-selected (default nudge)
├── Show savings: "Save 20% with annual billing"
├── Monthly price visible but de-emphasized
└── Consider: Show monthly cost of annual plan
```

## 7. Cognitive Biases Catalog

### 7.1 Top 20 Cognitive Biases (with Product Applications)

| # | Bias | Description | Product Application |
|---|------|-------------|---------------------|
| 1 | **Anchoring** | Over-relying on first piece of information | Show high price first, then actual price |
| 2 | **Loss Aversion** | Losses feel 2.5x worse than equivalent gains | "Don't lose your progress" > "Keep building" |
| 3 | **Social Proof** | Following what others do when uncertain | "50,000 teams trust us", user counts, testimonials |
| 4 | **Scarcity** | Valuing things more when they're rare | "Only 3 spots left", limited-time offers |
| 5 | **Reciprocity** | Feeling obligated to return favors | Free value first (content, tools), then ask |
| 6 | **Status Quo Bias** | Preferring current state (defaults win) | Pre-select recommended options |
| 7 | **Bandwagon Effect** | Doing what's popular | "Trending", "Most popular", usage numbers |
| 8 | **Sunk Cost Fallacy** | Continuing due to past investment | Progress bars, streaks, invested time displays |
| 9 | **Framing Effect** | Decisions change based on presentation | "$1/day" vs "$365/year" |
| 10 | **Confirmation Bias** | Seeking info that confirms beliefs | Personalized content that reinforces engagement |
| 11 | **Peak-End Rule** | Judging experience by peak and end moments | Make onboarding peak great, exit smooth |
| 12 | **IKEA Effect** | Overvaluing things we help create | User customization, profile completion |
| 13 | **Endowment Effect** | Overvaluing what we already have | Free trials (own it, then lose it) |
| 14 | **Availability Heuristic** | Judging likelihood by ease of recall | Recent success stories, vivid case studies |
| 15 | **Paradox of Choice** | Too many options = paralysis | Limit options, use recommendations |
| 16 | **Hyperbolic Discounting** | Preferring immediate rewards over larger future ones | Instant gratification in onboarding, quick wins |
| 17 | **Authority Bias** | Trusting experts and authority figures | Expert endorsements, certifications, credentials |
| 18 | **Mere Exposure Effect** | Preferring familiar things | Consistent branding, retargeting, content frequency |
| 19 | **Dunning-Kruger Effect** | Overestimating ability when inexperienced | Guided experiences, progressive disclosure |
| 20 | **Zeigarnik Effect** | Remembering uncompleted tasks more | Progress bars, incomplete profiles, open loops |

### 7.2 Bias Categories

```
Decision-Making Biases:
├── Anchoring, Framing, Loss Aversion, Sunk Cost
├── Affect when: Making purchases, evaluating options
└── Design strategy: Frame choices to guide decisions

Social Biases:
├── Social Proof, Bandwagon, Authority, In-Group
├── Affect when: Uncertain, looking for validation
└── Design strategy: Show social signals prominently

Memory Biases:
├── Peak-End Rule, Availability, Recency, Primacy
├── Affect when: Recalling experiences, forming opinions
└── Design strategy: Make key moments memorable

Self-Assessment Biases:
├── Dunning-Kruger, Overconfidence, Self-Serving
├── Affect when: Evaluating own performance
└── Design strategy: Provide honest feedback gently
```

## 8. Persuasion Principles

### 8.1 Cialdini's 7 Principles of Persuasion

```
1. RECIPROCITY
   "Give before you ask"
   ├── Free content, tools, templates
   ├── Unexpected generosity
   └── The gift should be meaningful, personal, unexpected

2. COMMITMENT & CONSISTENCY
   "Start with small yeses"
   ├── Get micro-commitments first
   ├── Public commitments are stronger
   └── Written commitments strongest of all

3. SOCIAL PROOF
   "Others are doing it"
   ├── Testimonials with specific results
   ├── User counts and activity metrics
   ├── "People like you" are using this
   └── Expert endorsements

4. AUTHORITY
   "Experts say so"
   ├── Credentials and expertise
   ├── Awards and certifications
   ├── Published research and data
   └── Uniforms and titles (even symbolic)

5. LIKING
   "We like people who like us"
   ├── Similarity (shared values, background)
   ├── Compliments (genuine appreciation)
   ├── Cooperation (working toward shared goals)
   └── Association (connected to positive things)

6. SCARCITY
   "Limited availability"
   ├── Time-limited offers
   ├── Quantity-limited offers
   ├── Exclusive access
   └── Fear of missing out (FOMO)

7. UNITY
   "We are the same tribe"
   ├── Shared identity ("fellow developers")
   ├── Shared experiences ("we've all been there")
   ├── Family and kinship language
   └── Co-creation (involve them in building)
```

### 8.2 Ethical Persuasion Framework

```
Ethical Check for Each Persuasion Technique:

1. TRANSPARENCY: Would the user agree with this technique if they knew about it?
2. BENEFIT: Does this genuinely help the user achieve their goals?
3. AUTONOMY: Can the user easily choose otherwise?
4. TRUTH: Are we being honest about what we're offering?
5. PROPORTIONALITY: Is the persuasion proportional to the stakes?

Red Lines (Never Cross):
├── Dark patterns (hiding unsubscribe, trick questions)
├── Manufactured scarcity (fake countdown timers)
├── Fake social proof (generated testimonials)
├── Exploiting vulnerable populations
├── Making it intentionally difficult to leave
└── Hidden costs or terms
```

## 9. Behavioral Design for Products

### 9.1 Onboarding Behavioral Design

```
Onboarding Framework: REDUCE

R - Reduce steps (minimum viable onboarding)
E - Earn a quick win (immediate value moment)
D - Default to ideal settings (smart defaults)
U - Use progressive disclosure (don't overwhelm)
C - Celebrate completion (emotional reward)
E - Enable habits early (first hook cycle)

First Session Goals:
├── Time to value < 5 minutes
├── One "aha moment" experienced
├── Core action completed at least once
├── Profile investment made (stored value)
├── Notification permission granted
└── Return trigger installed (email, push)

Onboarding Metrics:
├── Completion rate (each step)
├── Time to first value action
├── Day 1 retention
├── Day 7 retention
├── Setup completion rate
└── Feature adoption rate (key features)
```

### 9.2 Engagement Loops

```
Daily Engagement Loop:
Trigger → Action → Variable Reward → Investment → [Next Trigger]

Example (Fitness App):
Morning notification (trigger)
→ Log breakfast (action - easy)
→ See calorie count + streak (variable reward - mastery)
→ Set lunch reminder (investment - loads next trigger)
→ Lunch notification (next trigger)

Weekly Engagement Loop:
End-of-week summary (trigger)
→ Review progress (action)
→ See improvement + comparison to peers (reward)
→ Set next week's goals (investment)
→ Monday motivation email (next trigger)

Monthly Engagement Loop:
Monthly report (trigger)
→ Review achievements (action)
→ Milestone badge + social share (reward)
→ Upgrade consideration (investment)
→ New month challenge (next trigger)
```

### 9.3 Retention Behavioral Strategies

```
Retention = Habit Strength + Stored Value + Social Connection

Habit Strength:
├── Consistent triggers (same time, same context)
├── Easy actions (reduce friction over time)
├── Variable rewards (keep it interesting)
├── Celebrations (maintain positive emotion)
└── Streaks and consistency tracking

Stored Value:
├── User-generated content
├── Customized settings and preferences
├── History and progress data
├── Social connections and networks
├── Reputation and achievements
└── Integrated workflows

Social Connection:
├── Community features
├── Collaborative functionality
├── Social accountability
├── Shared experiences
├── Team-based goals

Anti-Churn Behavioral Triggers:
├── "You'll lose your X-day streak" (loss aversion)
├── "Your team is counting on you" (social commitment)
├── "You've made X progress" (sunk cost + endowment)
├── "New feature you've been wanting" (anticipation)
└── "People similar to you got Y result" (social proof)
```

## 10. Decision Frameworks & Checklists

### 10.1 Behavioral Audit Checklist

```
For each user action you want to increase:

Motivation Assessment:
- [ ] Is the benefit clearly communicated?
- [ ] Is the emotional appeal present?
- [ ] Is social motivation leveraged?
- [ ] Is the "why" more compelling than the effort?

Ability Assessment:
- [ ] Can the action be completed in < 30 seconds?
- [ ] Is the cognitive load minimal?
- [ ] Are there fewer than 3 steps?
- [ ] Is the interface intuitive (no instructions needed)?
- [ ] Is the action compatible with existing habits?

Prompt Assessment:
- [ ] Is there a clear, visible trigger?
- [ ] Is the trigger timed to a moment of readiness?
- [ ] Is the trigger in the user's natural flow?
- [ ] Does the trigger specify exactly what to do?
- [ ] Can the trigger be personalized?
```

### 10.2 Hook Model Checklist

```
For each feature/product:

Trigger:
- [ ] Internal trigger identified (what emotion/situation)
- [ ] External triggers designed for each stage
- [ ] Trigger-to-action path has < 3 steps
- [ ] Trigger frequency matches desired usage

Action:
- [ ] Simplest possible version defined
- [ ] Takes < 5 seconds
- [ ] Requires minimal cognitive effort
- [ ] Works on all devices/contexts

Variable Reward:
- [ ] Reward type identified (Tribe/Hunt/Self)
- [ ] Variability built in (not predictable)
- [ ] Reward satisfies the internal trigger
- [ ] Reward creates desire for more

Investment:
- [ ] User inputs stored value
- [ ] Investment improves next experience
- [ ] Investment happens after reward
- [ ] Investment loads the next trigger
```

### 10.3 Nudge Design Template

```
Nudge Design Canvas:

1. BEHAVIOR TARGET
   What behavior do we want? _______________
   What behavior exists now? _______________

2. BARRIER ANALYSIS

   - [ ] Transparent if questioned
```

### 10.4 Cognitive Bias Application Matrix

```
For each product touchpoint, consider:

Acquisition (Getting Users):
├── Social Proof: Show user numbers, testimonials
├── Authority: Expert endorsements, press logos
├── Scarcity: Limited-time offer, exclusive access
└── Reciprocity: Free value before asking

Activation (First Value):
├── Status Quo: Smart defaults, pre-configured
├── Peak-End Rule: Make first experience memorable
├── Zeigarnik: Show incomplete profile/setup
└── IKEA Effect: Let them customize early

Retention (Keep Coming Back):
├── Sunk Cost: Show invested time/progress
├── Loss Aversion: Streak protection, data loss warnings
├── Endowment: Free trial that becomes permanent
└── Commitment: Small commitments that escalate

Revenue (Monetization):
├── Anchoring: Show expensive option first
├── Framing: Per-day pricing, savings calculation
├── Decoy: Third option that makes target attractive
└── Scarcity: Time-limited pricing

Referral (Spreading):
├── Reciprocity: Give referrer AND referee value
├── Social Proof: "X friends already use this"
├── Unity: "Build your team/tribe"
└── Commitment: Public sharing = stronger commitment
```
